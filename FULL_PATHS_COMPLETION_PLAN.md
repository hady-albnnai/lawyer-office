## 0) الخلاصة التنفيذية

### أين نحن الآن؟
التطبيق **ليس هيكلًا فارغًا**. المسارات الأساسية التالية تعمل بمستوى تشغيلي جيد:

| المسار | الوضع الحالي |
|---|---|
| أول تشغيل `/setup` | ✅ حقيقي |
| لوحة اليوم `/today` | ✅ قراءة من DB |
| عمل جديد `/new-work` | ✅ يفتح شاشات إنشاء |
| أوامر العمل | ✅ دورة كاملة نسبيًا (DB+PDF+واتساب+اعتماد+أتمتة) |
| قوائم: دعاوى/ملفات/مستندات/أشخاص | ✅ مربوطة DB للقراءة + بذر اختياري |
| مالية / مكتبة / إعدادات / نسخ | ✅ مربوطة |
| بحث وتقارير | ⚠️ جزئي (محرك + مصادر مختلطة) |
| CI/Tests | ✅ |

### ماذا يعني «كل المسارات»؟
ليس فقط «الشاشة تفتح»، بل:

```text
UI → Validation → Repository → SQLite
  → Timeline/Activity/Deficiencies/Tasks (حسب الحدث)
  → UI refresh
  → لا Success كاذب
```

### تقدير الإنجاز الكلي
- **تشغيلي يومي (v1.0):** ~85–90%
- **مطابقة المخطط الكامل end-to-end:** ~75–80%
- **المتبقي لإغلاق 100% مسارات:** الخطة أدناه (مراحل F1→F8)

## 1) تعريف الاكتمال النهائي (Definition of Done)

لا تُعلن «كل المسارات تعمل» إلا بعد:

1. **لا Seed في واجهة الإنتاج** إلا عبر خيار أول تشغيل الصريح.
2. **كل زر حفظ/إنشاء/تعديل/حذف** يكتب DB أو يفشل برسالة واضحة.
3. **كل قائمة رئيسية** تُقرأ من Repository/Stream حقيقي.
4. **كل شاشة تفاصيل** تعرض بيانات الكيان من DB (لا تركيب من list seed).
5. **دورة النتيجة اليومية** (جلسة/مراجعة/مهمة) تحدّث:
   - حالة المهمة/الجلسة
   - الموعد القادم (إن وجد)
   - timeline
   - activity log
   - deficiencies (فتح/إغلاق)
   - مصروف (إن وُجد)
6. **الشركات/العقود/الإجراءات** لها: قائمة + إنشاء + تفاصيل + حفظ DB.
7. **الملفات** تفتح تفاصيل الكيان الصحيح (دعوى/عقد/شركة/إجراء).
8. **البحث** يفهرس من DB الحي (دعاوى/عقود/شركات/إجراءات/أشخاص/وكالات/مستندات/أوامر/مالية/مكتبة).
9. **اختبارات مسار كامل** (unit + data-layer + سيناريوهات حرجة).
10. **اختبار يدوي Windows** لسيناريو يوم محامٍ كامل + build release.
