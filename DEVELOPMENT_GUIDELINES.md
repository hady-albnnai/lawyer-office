### المرحلة 4: الرفع إلى GitHub

#### قبل الرفع
- [ ] مراجعة الكود نهائياً
- [ ] التأكد من عدم وجود أخطاء
- [ ] التأكد من دعم RTL
- [ ] التأكد من الثيم الموحد
- [ ] التأكد من التوثيق الكامل

#### عملية الرفع
- [ ] إضافة الملفات إلى Git
- [ ] كتابة Commit Message واضحة
- [ ] Push إلى الفرع main
- [ ] التحقق من الرفع على GitHub

#### متطلبات Commit Message
- [ ] استخدام prefixed commit messages:
  - `feat:` - لإضافة ميزة جديدة
  - `fix:` - لإصلاح خطأ
  - `docs:` - لتحديث التوثيق
  - `style:` - لتعديلات على الأسلوب
  - `refactor:` - لإعادة هيكلة الكود
  - `perf:` - لتحسين الأداء
  - `test:` - لإضافة اختبارات
  - `chore:` - للمهام الإدارية

- [ ] مثال: `feat(cases): add create case wizard with 8 steps`

## 🎨 المعايير الفنية

### 1. الثيم الموحد

#### استخدام الألوان
```dart
// يجب استخدام AppColors من app_colors.dart
import '../../theme/app_colors.dart';

Color primaryColor = AppColors.primaryNavy;
Color secondaryColor = AppColors.secondaryGold;
Color backgroundColor = AppColors.backgroundLight;
```

#### استخدام أنماط النصوص
```dart
// يجب استخدام AppTextStyles من app_text_styles.dart
import '../../theme/app_text_styles.dart';

Text(
  'مرحبا',
  style: AppTextStyles.headline1,
)
```

#### استخدام الثيم
```dart
// يجب استخدام AppTheme.lightTheme من app_theme.dart
import '../../theme/app_theme.dart';

MaterialApp(
  theme: AppTheme.lightTheme,
  // ...
)
```

### 2. دعم RTL

#### في الواجهات
- [ ] استخدام `Directionality` إذا لزم الأمر
- [ ] استخدام `TextDirection.rtl` للنصوص العربية
- [ ] التأكد من أن جميع النصوص العربية تظهر بشكل صحيح

#### في Models
- [ ] دعم الحقول العربية
- [ ] دعم الترجمات

#### في الشاشات
- [ ] اختبار جميع الشاشات مع RTL
- [ ] التأكد من أن الأيقونات تظهر بشكل صحيح
- [ ] التأكد من أن التصفح يعمل بشكل صحيح

### 3. هيكل الملفات

#### قواعد التسمية
- [ ] استخدام snake_case لاسماء الملفات
- [ ] استخدام PascalCase لاسماء Classes
- [ ] استخدام camelCase لاسماء Functions و Variables
- [ ] استخدام UPPER_CASE لاسماء Constants

#### تنظيم الملفات
```

│   ├── theme/
│   ├── widgets/
```

## 📝 قالب Commit Message

```
<type>(<scope>): <description>

[optional body]

[optional footer]
```

**أمثلة:**
```
feat(cases): add create case wizard with 8 steps
fix(theme): correct AppColors values
Docs(readme): update project documentation
style(sidebar): improve badge alignment
refactor(models): reorganize case models
```
