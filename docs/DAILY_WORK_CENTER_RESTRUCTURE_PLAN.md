# خطة إعادة هيكلة مكتب العمل اليومي في ميزان

> **اسم المرحلة:** مكتب العمل اليومي — Daily Work Center  
> **النطاق:** لوحة اليوم، لوحة الغد، الأسبوع، التقويم، إدخال النتائج، توزيع العمل، طباعة لوائح العمل، المهام المخصصة، وربط كل ذلك بالصلاحيات وسجل المسؤولية.  
> **الحالة:** خطة هيكلية معتمدة تُنفّذ بعد إغلاق طبقة الصلاحيات والتدقيق، وتُستخدم قبل بناء المساعد الذكي.

---

## 1. الفكرة الأساسية

الاسم الحالي مثل:

```text
لوحة اليوم
```

لا يكفي لوصف طريقة عمل المحامي. المطلوب هو:

```text
مكتب العمل
```

أو:

```text
مكتب العمل اليومي
```

لأنه ليس مجرد Dashboard، بل مركز تشغيل يومي للمكتب.

هذا المركز يجيب عن أسئلة المحامي العملية:

```text
ماذا حدث اليوم ويحتاج نتيجة؟
ماذا عندي غداً لأحضّره وأوزعه؟
ما الأعمال القادمة خلال الأسبوع؟
من المكلّف بكل عمل؟
ما الملفات التي تحتاج انتباه؟
ما الذي يجب طباعته وتوزيعه على الفريق؟
```

### الاسم الرئيسي


### التبويبات الرئيسية

### الهدف من كل تبويب

```text

التقويم: عرض زمني تفاعلي للأعمال
```

## 3. فلسفة اليوم والغد

### 3.1 اليوم

تبويب اليوم ليس فقط لعرض المواعيد، بل هدفه الأساسي:

```text
إدخال نتائج أعمال اليوم
```

أمثلة:

```text
جلسة اليوم → ماذا حدث؟ ما الموعد القادم؟ ما المطلوب؟
إجراء اليوم → هل تم؟ ما الخطوة التالية؟
أمر عمل اليوم → هل عاد المعقب بنتيجة؟
نقص اليوم → هل تم استكماله؟
مهمة مخصصة اليوم → هل أُنجزت أو تُرحّل؟
```

### 3.2 الغد

تبويب الغد هو الأهم للتحضير:

```text
تحضير وتوزيع عمل الغد
```

المحامي غالباً لا يخطط لليوم فقط، بل يحضر للغد:

```text
تحضير ملف جلسة الغد
توزيع مراجعة الدوائر
إرسال المعقب
تجهيز مذكرة
تأكيد حضور الموكل
مراجعة النواقص قبل الجلسة
```

لذلك يجب أن يكون تبويب الغد واضحاً وقابلاً للطباعة والتوزيع.

---

## 4. قاعدة العرض الأساسية

أي شيء تاريخه هو تاريخ التبويب المعروض يجب أن يظهر إذا كان المستخدم يملك صلاحية رؤيته.

مثلاً في تبويب اليوم:

```text
كل عمل تاريخه اليوم يظهر
```

وفي تبويب الغد:

```text
كل عمل تاريخه غداً يظهر
```

لكن مع احترام الصلاحيات:

```text
المالية لا تظهر لمن لا يملك finance.view
أوامر العمل لا تظهر لمن لا يملك work_orders.view
المستندات لا تفتح لمن لا يملك documents.open
سجل المسؤولية لا يظهر لمن لا يملك audit.view
```

## 8. ترتيب العرض داخل مكتب العمل

لا تعرض العناصر كقائمة عشوائية. الترتيب المقترح:

```text
1. يحتاج انتباه
2. جلسات
3. تحضيرات ومراجعات
4. أوامر العمل
5. النواقص
6. مهام مخصصة
7. تذكيرات العقود والمالية
8. ملفات مستوردة تحتاج استكمال
```

القسم لا يظهر إذا:

```text
لا يوجد عناصر فيه
أو المستخدم لا يملك صلاحية رؤيته
```

---

## 9. قسم يحتاج انتباه

قسم مهم يظهر في أعلى مكتب العمل.

يعرض:

```text
جلسة اليوم بلا وكالة
جلسة اليوم بلا مستند مطلوب
أمر عمل متأخر
أمر عمل بانتظار اعتماد
دعوى جارية مستوردة بلا جلسة قادمة
إجراء جارٍ بلا خطوة تالية
نقص حرج
تذكير مالي مهم إن كان المستخدم يملك صلاحية المالية
```

## 10. بطاقة عنصر العمل

كل بطاقة يجب أن تكون مفهومة للمحامي.

### مثال بطاقة جلسة

```text
جلسة — عالية
دعوى تعويض أحمد الخطيب
محكمة البداية المدنية — أساس 123/2024
الوقت: 09:00
المكلف: المحامي سامر
المطلوب: تقديم مذكرة

[فتح الملف] [إدخال نتيجة] [المزيد]
```

### مثال بطاقة أمر عمل

```text
أمر عمل — تصوير ضبط
الملف: دعوى أحمد الخطيب
المكلف: المعقب أبو علي
موعد التنفيذ: غداً
الحالة: بانتظار نتيجة

[فتح الأمر] [إرسال واتساب] [نتيجة]
```

## 11. إدخال النتيجة من تبويب اليوم

في تبويب اليوم، كل عنصر مستحق اليوم يجب أن يملك زر:

```text
إدخال نتيجة
```

لكن نافذة النتيجة تختلف حسب نوع العنصر.

---

## 12. نتيجة الجلسة

نافذة نتيجة الجلسة يجب أن تسأل:

```text
ماذا حدث في الجلسة؟
النتيجة
الموعد القادم
ما المطلوب قبل الموعد القادم؟
هل تريد تذكيراً قبل الموعد؟
هل تحتاج أمر عمل؟
هل توجد مرفقات؟
```

### النتائج المقترحة

```text
تأجيل
حجز للحكم
صدور قرار
شطب
متابعة
إبراز مذكرة
إبراز وثيقة
أخرى
```

### الترحيل التلقائي

عند إدخال الموعد القادم:

```text
ينشأ عنصر عمل جديد بتاريخ الموعد القادم
```

إذا أضاف المستخدم تذكيراً تحضيرياً:

```text
ينشأ عنصر تحضير بتاريخ قبل الموعد القادم
```

مثال:

```text
الجلسة القادمة: 2026-08-20
تذكير تحضيري قبل 5 أيام
→ مهمة تحضير بتاريخ 2026-08-15
```

## 15. نتيجة النقص

تسأل:

```text

هل توجد مرفقات؟

```

إذا تم:

```text

```

## 16. المهام المخصصة

### 16.1 لماذا نحتاجها؟

مهام المحامي لا يمكن حصرها بكل الأنواع النظامية.

أمثلة:

```text
اتصال بموكل
مقابلة شاهد
تحضير مذكرة
مراجعة ملف
متابعة خبير
اجتماع داخلي
تدقيق مستندات
مراجعة بريد
زيارة محكمة غير مرتبطة بملف محدد
```

لذلك نضيف:

```text
مهمة مخصصة
```

---

## 17. قوالب المهام المخصصة

لا تكون المهمة المخصصة مجرد نص حر فقط. الأفضل دعم:

```text
قوالب مهام
```

مثال قالب:

```text
اتصال بموكل
```

نتائجه:

```text
تم الاتصال
لم يرد
طلب موعد
أرسل مستندات
يحتاج متابعة لاحقة
```

مثال آخر:

```text
تحضير مذكرة
```

نتائجه:

```text
تم التحضير
تحتاج مراجعة الأستاذ
تحتاج مستندات
مؤجلة
```

## 20. إضافة عمل لليوم

في مكتب العمل يوجد زر:

```text
إضافة عمل
```

لا يفتح نموذجاً واحداً، بل يعرض خيارات حسب الصلاحيات:

```text
دعوى جديدة
جلسة لملف موجود
إجراء إداري
شركة
عقد
أمر عمل
مهمة مخصصة
```

الأعمال النظامية تفتح شاشاتها الرسمية:

```text
دعوى جديدة → معالج إنشاء الدعوى
شركة → معالج تأسيس الشركة
إجراء → شاشة الإجراء الإداري
أمر عمل → نافذة أمر العمل
```

أما:

```text
مهمة مخصصة
```

فتفتح نموذج المهمة المخصصة.

## 22. الغد والتحضير

تبويب الغد يركز على:

```text
توزيع العمل
تجهيز الملفات
تأكيد المكلفين
طباعة لائحة الغد
```

كل بطاقة في الغد تحتوي أزراراً مثل:

```text
تعيين مكلف
إضافة ملاحظة تحضير
إنشاء أمر عمل
طباعة المهمة
فتح الملف
```

---

## 23. الطباعة

يجب دعم:

```text
طباعة لائحة اليوم
طباعة لائحة الغد
طباعة الأسبوع
طباعة حسب المكلف
طباعة حسب النوع
إخفاء المالية
إظهار النواقص
```

لأن المكلف قد يكون:

```text
مستخدم داخلي
اسم خارجي
```

فالفلاتر تكون حسب:

```text
المكلف
```

وليس فقط المستخدم.


```text

إجراءات
أوامر عمل
نواقص
```

## 25. الربط مع الأرشيف

الملفات الجارية المستوردة من الأرشيف يجب أن تنتج عناصر عمل.

مثال:

```text
دعوى جارية مستوردة لها جلسة غداً
```

تظهر في:

```text
الغد > جلسات الغد
```

مثال:

```text
دعوى جارية بلا وكالة
```

تظهر في:

```text
يحتاج انتباه
ملفات جارية تحتاج استكمال
```
