# تقرير اكتمال نظام الأرشيف القديم

تاريخ التوثيق: 2026-07-17

هذا التقرير يوثق حالة تنفيذ نظام الأرشيف القديم بعد سلسلة إعادة الهيكلة. الهدف من النظام أن يكون عملياً لمكتب محاماة حقيقي، وأن يفرّق بوضوح بين الملف القديم المنتهي والملف القديم الجاري الذي ما زال يؤثر على العمل اليومي.

## 1. القاعدة التشغيلية الأساسية

تم اعتماد القاعدة التالية في كل مسار الأرشيف:

```text
الأرشيف الجاري = ملف حي، مواعيده ونواقصه تنعكس على مكتب العمل والتقويم.
الأرشيف المنتهي = حفظ وبحث فقط، ولا يولد عملاً قادماً أو نواقص تشغيلية.
```

## 2. مداخل الأرشيف

أصبح الوصول إلى الأرشيف متاحاً من عدة أماكن:

- `ملفات المكتب > إدخال الأرشيف القديم`.
- `الملفات الجارية` مع تحديد تلقائي لأرشيف جارٍ.
- `الملفات المنتهية` مع تحديد تلقائي لأرشيف منتهٍ.
- شاشة `عمل جديد` عبر:
  - إدخال أرشيف جارٍ.
  - إدخال أرشيف منتهٍ.
- زر `إضافة سريع` في الشريط العلوي.
- `مكتب العمل` لإدخال أرشيف جارٍ فقط.
- أوامر البحث الشامل:
  - `/ارشيف جاري`.
  - `/ارشيف منتهي`.

## 3. معالج إدخال الأرشيف

تم تنفيذ معالج هرمي يبدأ بـ:

```text
أرشيف جارٍ / أرشيف منتهٍ
```

ثم:

```text
دعوى / شركة / إجراء / عقد / وكالة / أرشيف غير محدد
```

ثم التصنيفات التفصيلية حسب نوع الملف.

### الدعاوى

يدعم تصنيفات مثل:

- مدنية.
- جزائية.
- تجارية.
- شرعية.
- إدارية.

وتظهر درجات ومحاكم مناسبة مثل:

- صلح.
- بداية.
- استئناف.
- نقض.
- مخاصمة.
- نيابة عامة / تحقيق / إحالة / جنايات حسب الجزائي.

### الشركات

يدعم:

- شركات أشخاص.
- شركات أموال.
- تضامنية.
- توصية بسيطة.
- محدودة المسؤولية.
- مساهمة مغفلة خاصة.
- مساهمة مغفلة عامة.

### الإجراءات والعقود والوكالات

تم تطبيق نفس فلسفة التصنيف المرن عليها، مع إمكانية إضافة أنواع غير معروفة مسبقاً.

## 4. التصنيفات المخصصة

تم تنفيذ نظام تصنيفات مخصصة دائم عبر جدول SQL مستقل:

```text
archive_reference_values
```

يدعم:

- إضافة تصنيف جديد.
- حفظه لاستخدامه لاحقاً.
- البحث ضمن التصنيفات.
- تعديل التصنيف.
- تعطيله دون حذف تاريخ استخدامه.
- إعادة تفعيله.
- تصدير التصنيفات CSV.

## 5. دفعات الأرشيف

يدعم النظام دفعات:

- أرشيف ورقي.
- أرشيف إلكتروني.
- Excel / CSV.
- أرشيف مختلط.

مع إدارة كاملة للدفعات:

- إنشاء دفعة.
- حفظ مصدر الدفعة.
- تعديل اسم ومصدر وملاحظات الدفعة.
- فتح مصدر الدفعة.
- البحث في الدفعات.
- فلترة الدفعات حسب المصدر والحالة.
- مؤشرات تقدم مراجعة واعتماد.
- نسخ ملخص الدفعة.
- تصدير عناصر الدفعة CSV.

## 6. استيراد الملفات والمجلدات

يدعم:

- استيراد ملفات متعددة.
- استيراد مجلد كامل بشكل recursive.
- حساب SHA-256.
- كشف المكررات.
- حفظ الملفات غير المكررة فقط.
- تسجيل العناصر الفاشلة.
- فتح الدفعة مباشرة بعد الاستيراد.

## 7. استيراد CSV الآمن

تم تنفيذ مسار آمن:

```text
CSV → معاينة → تنبيهات جودة → تأكيد → عناصر مراجعة
```

يدعم:

- UTF-8 BOM.
- الفواصل `,` و`;` و Tab.
- معاينة الأعمدة والصفوف.
- تنبيهات جودة.
- كشف الصفوف المكررة داخل الملف.
- كشف الصفوف المستوردة سابقاً.
- تحويل الصفوف إلى عناصر أرشيف للمراجعة، وليس سجلات تشغيلية مباشرة.
- ربط صف CSV لاحقاً كوثيقة بياناتية بلا ملف مرفق.

## 8. صندوق غير مصنف

يدعم:

- عرض العناصر التي تحتاج مراجعة.
- بحث وفلاتر.
- تفاصيل كل عنصر.
- فتح الملف المستورد.
- فتح المصدر الأصلي.
- ربط العنصر بملف مكتب.
- اعتماد عام.
- رفض.
- إعادة عنصر مرفوض إلى المراجعة.
- تصدير المعروض CSV.

## 9. المكررات

يدعم:

- عرض المكررات.
- البحث في المكررات.
- تفاصيل المكرر.
- فتح المصدر الأصلي.
- فتح الملف المحفوظ إن وجد.
- مقارنة المكرر مع الأصل جنباً إلى جنب.
- تجاهل المكرر مع ملاحظة مراجعة.
- تصدير المكررات CSV.

## 10. ربط الأرشيف بملفات المكتب

يمكن ربط عنصر أرشيف مع:

- دعوى.
- إجراء إداري.
- شركة.
- عقد.
- شخص / جهة.
- وكالة.

وعند الربط:

- يتم إنشاء مستند.
- يتم إنشاء DocumentLink.
- يتم تحديث حالة عنصر الأرشيف إلى معتمد.
- يتم تسجيل Timeline.
- يتم تسجيل Audit.
- يمكن إدخال نوع مستند مخصص.
- يمكن إدخال بيانات الأصل الورقي أثناء الربط.

## 11. الأرشيف الورقي الرسمي

تم إنشاء جدول رسمي:

```text
document_paper_metadata
```

يحفظ:

- هل الأصل محفوظ.
- مكان الأصل.
- الصندوق.
- الرف.
- المجلد الورقي.
- هل يجوز إتلاف الأصل.
- من راجع النسخة الرقمية.
- تاريخ المراجعة.
- ملاحظات.

ويدعم:

- ترحيل تلقائي من الملاحظات القديمة.
- كشف الأرشيف الورقي.
- البحث والفلترة.
- تعديل بيانات الأصل الورقي.
- تعليم فردي أو جماعي كمراجع رقمياً.
- نسخ بيانات أصل ورقي.
- فتح المستند المرتبط.
- تصدير كشف CSV.

## 12. ملفات المكتب

تم ضبط ملفات المكتب حسب القرار النهائي:

```text
الملفات الجارية
الملفات المنتهية
إدخال الأرشيف القديم
```

مع:

- فلاتر نوع الملف داخل الشاشة.
- منع إظهار نواقص تشغيلية على الملفات المنتهية.
- إظهار الملفات الجارية التي تحتاج استكمال.
- تصدير ملفات الاستكمال CSV.
- نسخ مطلوبات الاستكمال لملف معين.

## 13. أثر الأرشيف على مكتب العمل

تم ضبط الأثر كما يلي:

- الدعوى الجارية ذات موعد قادم تولد مهمة في مكتب العمل.
- الإجراء الجاري ذو موعد متابعة يولد مهمة.
- الشركة الجارية تولد مرحلة متابعة.
- العقد الجاري يمكن أن يولد تذكير متابعة.
- الأرشيف المنتهي لا يولد مواعيد ولا نواقص ولا Checklist ولا مراحل تأسيس.

## 14. المستندات والعارض

تم تحسين المستندات:

- عرض بيانات الأصل الورقي.
- عرض الموقع الكامل.
- فتح الملف المرتبط.
- دعم مستندات بياناتية بلا ملف، مثل صفوف CSV المرتبطة.
- عرض بيانات صف CSV بشكل منظم.
- تعليم كمراجع من عارض المستند.
- تعديل بيانات الأصل الورقي من عارض المستند.

## 15. التصديرات

تم تنفيذ تصديرات CSV مع دعم العربية في Excel:

- تقرير الجودة.
- عناصر الدفعة.
- صندوق غير مصنف.
- المكررات.
- ملفات الاستكمال.
- الأرشيف الورقي.
- التصنيفات المخصصة.
- قوالب الاستيراد.

## 16. سجل المسؤولية Audit

تم تسجيل العمليات الحساسة، ومنها:

- إنشاء دفعة.
- استيراد ملفات أو مجلدات أو CSV.
- عرض التفاصيل.
- فتح الملف أو المصدر.
- الربط والاعتماد.
- الرفض والتجاهل.
- إعادة المراجعة.
- تصدير الكشوف.
- تعديل التصنيفات.
- تعديل بيانات الأصل الورقي.

## 17. المتبقي بعد اكتمال هذه المرحلة

المتبقي ليس بناء الأرشيف نفسه، بل التحقق والتثبيت:

1. تشغيل التطبيق على Windows وفحص البناء.
2. إصلاح أي أخطاء Compile تظهر بسبب عدم توفر Flutter في بيئة التنفيذ هنا.
3. اختبار سيناريوهات الأرشيف عملياً ببيانات حقيقية.
4. لاحقاً، إذا أراد المكتب، يمكن بناء مرحلة متقدمة تنشئ سجلات تشغيلية مباشرة من CSV بعد اعتماد قواعد مطابقة صارمة، لكن المسار الحالي الآمن مكتمل.

## الخلاصة

نظام الأرشيف الآن مكتمل كمرحلة عملية متقدمة:

```text
إدخال → تصنيف → رفع/استيراد → مراجعة → مكررات → ربط → أصل ورقي → ملفات مكتب → مكتب عمل → تقارير وتصدير
```

وهو مصمم ليحافظ على الفرق الجوهري بين:

```text
ملف جارٍ حي
ملف منتهٍ محفوظ
```
