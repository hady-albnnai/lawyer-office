# خطة شاشة الملفات الجارية والمنتهية

تاريخ الاعتماد: 2026-07-20

هذه الخطة تضبط شاشة `ملفات المكتب` بعد تثبيت القاعدة التالية:

```text
الملفات الجارية = لوحة تشغيل يومية.
الملفات المنتهية = أرشيف بحث وإجراءات لاحقة.
إدخال الأرشيف القديم = مسار مستقل.
إنشاء الجديد = من مكتب العمل فقط.
```

لا تحتوي شاشتا الملفات الجارية والمنتهية على إنشاء ملف جديد. فتح ملف جديد يتم فقط من:

```text
مكتب العمل > إنشاء جديد
```

أما إدخال الملفات القديمة فيتم فقط من:

```text
ملفات المكتب > إدخال الأرشيف القديم
```

---

## 1. قاعدة ملفات المكتب

قسم `ملفات المكتب` يحتوي عملياً على:

```text
الملفات الجارية
الملفات المنتهية
إدخال الأرشيف القديم
```

ولا يفرش أنواع الملفات في الشريط الجانبي كأقسام مستقلة.

أنواع الملفات تظهر داخل الشاشة كفلاتر وتصنيفات:

```text
دعاوى
إجراءات / معاملات
عقود
شركات
وكالات
```

---

## 2. الفرق بين الملفات الجارية والمنتهية

### 2.1 الملفات الجارية

هي الملفات التي ما زال المكتب يعمل عليها.

تؤثر على:

```text
مكتب العمل
التقويم
المواعيد
أوامر العمل
النواقص التشغيلية
المتابعة اليومية
```

أمثلة:

```text
دعوى لها جلسة قادمة
إجراء إداري بانتظار مراجعة
عقد قيد التفاوض
شركة قيد التأسيس أو التعديل
وكالة تحتاج استكمال أو مرتبطة بملف حي
```

### 2.2 الملفات المنتهية

هي ملفات محفوظة للبحث والطباعة والرجوع.

لا تؤثر على:

```text
مكتب العمل
التقويم
النواقص التشغيلية اليومية
```

لكن قد تحتوي:

```text
إجراءات لاحقة
طعون
تجديد
لا مانع
محضر تسليم مستندات
مالية متبقية
أصل ورقي
```

أمثلة:

```text
دعوى انتهت بحكم
دعوى انتهت بصلح
دعوى مشطوبة قابلة للتجديد
ملف تم التخلي عنه
عقد منتهٍ
شركة مؤرشفة
وكالة ملغاة أو منتهية
```

## 3. شاشة الملفات الجارية

### 3.1 العنوان والوصف

العنوان:

```text
الملفات الجارية
```

الوصف:

```text
الملفات التي ما زال المكتب يعمل عليها وتؤثر على المواعيد ومكتب العمل.
```

### 3.2 المؤشرات العلوية

تعرض أعلى الشاشة كبطاقات صغيرة:

```text
إجمالي الملفات الجارية
دعاوى
إجراءات / معاملات
عقود
شركات
وكالات
متأخرة
لديها موعد قريب
تحتاج استكمال
لديها أوامر عمل مفتوحة
تعارض محتمل
```

إيضاحات:

```text
متأخرة: ملفات لها موعد أو إجراء تجاوز تاريخه.
لديها موعد قريب: ملفات لها جلسة أو مراجعة خلال مدة يحددها المكتب، مثل 7 أيام.
تحتاج استكمال: ملفات فيها نواقص تشغيلية مثل وكالة ناقصة أو رقم أساس غير مدخل أو مستند مطلوب.
تعارض محتمل: ملفات أنشئت مع وجود تحذير تعارض مصالح أو طرف سابق ولم يغلق التحذير بعد.
```

### 3.3 البحث

حقل البحث:

```text
ابحث في الملفات الجارية...
```

يبحث في:

```text
رقم ملف المكتب
اسم الموكل
اسم الخصم
عنوان الملف
موضوع الدعوى / العقد / الإجراء
المحكمة أو الجهة
رقم الأساس أو المرجع
رقم الوكالة
رقم العقد
اسم الشركة
اسم المحامي المسؤول
```

### 3.4 فلاتر نوع الملف

```text
كل الملفات
دعاوى
إجراءات / معاملات
عقود
شركات
وكالات
```

عند اختيار نوع ملف تتغير الفلاتر الفرعية المناسبة له.

### 3.5 الفلاتر التشغيلية العامة

```text
كل الحالات
لديها موعد قريب
متأخرة
تحتاج استكمال
لديها أوامر عمل مفتوحة
بانتظار نتيجة
بانتظار اعتماد
تعارض محتمل
وكالة ناقصة / غير مكتملة
أصل ورقي غير محدد
```

هذه الفلاتر تظهر للملفات الجارية فقط لأنها فلاتر تشغيلية.

### 3.6 فلاتر الدعاوى الجارية

حالات الدعوى:

```text
قيد التحضير
مسجلة
قيد التبليغ
قيد المحاكمة
بانتظار قرار
بانتظار تنفيذ
معلقة
مؤجلة
مشطوبة / قابلة للتجديد
```

نوع الدعوى:

```text
مدنية
تجارية
جزائية
أحوال شخصية
قضاء إداري
```

المرحلة:

```text
صلح مدني
بداية مدنية
استئناف مدني
نقض مدني
صلح جزاء
بداية جزاء
استئناف جنح
جنايات
نقض جزائي
محكمة قضاء إداري
مرحلة مخصصة
```

### 3.7 فلاتر الإجراءات / المعاملات الجارية

```text
قيد التحضير
مقدمة للجهة
بانتظار مراجعة
بانتظار موافقة
بانتظار استكمال نقص
بانتظار دفع رسم
منجزة جزئياً
معلقة
```

### 3.8 فلاتر العقود الجارية

```text
مسودة
قيد المراجعة
قيد التفاوض
بانتظار توقيع
موقع وقيد المتابعة
بانتظار تنفيذ التزام
معلق
```

### 3.9 فلاتر الشركات الجارية

```text
قيد التأسيس
بانتظار موافقة
بانتظار دفع رسوم
بانتظار توقيع الشركاء
قيد التسجيل
قيد تعديل لاحق
بانتظار نشر / استكمال
```

### 3.10 فلاتر الوكالات الجارية

```text
فعالة
غير مكتملة البيانات
بانتظار صورة السند
بانتظار تصديق / صورة
مرتبطة بملف جارٍ
تحتاج مراجعة بسبب عزل / اعتزال
```

## 4. بطاقة الملف الجاري

البطاقة موحدة في الشكل، وتتخصص بعض تفاصيلها حسب نوع الملف.

### 4.1 أعلى البطاقة

```text
رقم ملف المكتب
نوع الملف
التصنيف الفرعي
الحالة
```

أمثلة:

```text
2026/001 — دعوى — بداية مدنية — قيد المحاكمة
CON-2026-004 — عقد — بيع عقار — قيد التفاوض
```

### 4.2 عنوان البطاقة

```text
عنوان الملف / الموضوع المختصر
```

أمثلة:

```text
مطالبة مالية
إزالة شيوع
تثبيت بيع
عقد شراكة
تأسيس شركة محدودة المسؤولية
معاملة ترخيص
وكالة عامة للموكل أحمد
```

### 4.3 الأطراف الرئيسية

حسب نوع الملف:

```text
دعوى: الموكل الرئيسي، الخصم الرئيسي.
إجراء: صاحب العلاقة، الجهة الإدارية.
عقد: الأطراف الرئيسية.
شركة: اسم الشركة / الشركاء الرئيسيون.
وكالة: الموكل / الموكلون، المحامي / المحامون.
```

### 4.4 المرجع أو الجهة

حسب النوع:

```text
المحكمة
الجهة الإدارية
السجل التجاري
فرع النقابة
الطرف المتعاقد
```

### 4.5 رقم الأساس أو المرجع

يظهر فقط إذا له معنى:

```text
رقم الأساس وسنته
رقم الطعن
رقم المعاملة
رقم السجل التجاري
رقم الوكالة
رقم العقد الداخلي
```

مع منع الخلط بينه وبين:

```text
رقم ملف المكتب
```

### 4.6 الموعد القادم

يعرض:

```text
الموعد القادم
نوع الموعد
الإجراء القادم
```

أمثلة:

```text
جلسة — 2026/08/12 — إبراز مذكرة
مراجعة مالية — 2026/08/05 — دفع رسم
توقيع عقد — 2026/08/01
مراجعة فرع النقابة — 2026/08/03
```

إذا لا يوجد موعد:

```text
لا يوجد موعد قادم
```

وفي الملفات الجارية قد يظهر ذلك كنقص تحذيري حسب نوع الملف.

### 4.7 النواقص والتنبيهات

تعرض كوسوم صغيرة:

```text
لا توجد وكالة
رقم أساس غير مدخل
لا يوجد موعد قادم
مستند مطلوب غير مرفق
أصل ورقي غير محدد
تعارض محتمل
وكالة غير مكتملة
وكالة معزول عنها / اعتزال
أمر عمل متأخر
```

### 4.8 أوامر العمل

تعرض مختصراً:

```text
أوامر مفتوحة: 3
نتيجة تنتظر اعتماد: 1
تحتاج تصحيح: 1
```

### 4.9 أزرار البطاقة الجارية

الأزرار العامة:

```text
فتح الملف
إضافة أمر عمل
إضافة مستند
```

أزرار حسب النوع:

```text
دعوى: إدخال نتيجة جلسة، إضافة جلسة.
إجراء: تحديث خطوة، إضافة مراجعة.
عقد: تحديث حالة التفاوض، فتح نسخة العقد.
شركة: تحديث إجراء الشركة، إضافة تعديل.
وكالة: فتح الوكالة، ربط بملف، تسجيل عزل / اعتزال.
```

الأزرار المالية تظهر حسب الصلاحية:

```text
سند قبض
مصروف
```

## 5. شاشة الملفات المنتهية

### 5.1 العنوان والوصف

العنوان:

```text
الملفات المنتهية
```

الوصف:

```text
ملفات محفوظة للبحث والطباعة والرجوع، ولا تؤثر على مكتب العمل إلا عند فتح إجراء لاحق.
```

### 5.2 المؤشرات العلوية

```text
إجمالي الملفات المنتهية
دعاوى
إجراءات
عقود
شركات
وكالات
إغلاق إداري غير مكتمل
لها إجراءات لاحقة
لها أصول ورقية في المكتب
مالية متبقية
```

إيضاحات:

```text
إغلاق إداري غير مكتمل: الملف انتهى قضائياً أو عملياً، لكن بقيت أمور مكتبية مثل تسليم أصول أو تسوية مالية.
لها إجراءات لاحقة: ملفات يمكن أن تفتح تجديد أو طعن أو تنفيذ أو إعادة فتح.
مالية متبقية: يوجد رصيد أو مصروف أو أتعاب غير مسواة.
```

### 5.3 البحث

يبحث في:

```text
رقم ملف المكتب
الأطراف
الموضوع
المحكمة / الجهة
رقم الأساس / المرجع
سبب الإغلاق
تاريخ الإغلاق
مكان الأصل الورقي
رقم الوكالة
رقم العقد
اسم الشركة
```

### 5.4 فلاتر نوع الملف

```text
كل الملفات المنتهية
دعاوى
إجراءات / معاملات
عقود
شركات
وكالات
```

### 5.5 فلاتر عامة للمنتهية

```text
منتهية بإغلاق نهائي
منتهية بتسليم / تخلي
منتهية بصلح
منتهية بسبب آخر
إغلاق إداري مكتمل
إغلاق إداري غير مكتمل
لها أصل ورقي في المكتب
أصل ورقي غير محدد
مالية متبقية
```

### 5.6 فلاتر الدعاوى المنتهية

```text
منتهية بحكم
منتهية بقرار
منتهية بشطب
منتهية برد شكلي
منتهية بتنازل / إسقاط
منتهية بصلح
منتهية بانتهاء التنفيذ
منتهية بتخلي / تسليم
قابلة لتجديد
قابلة لاستئناف
قابلة لنقض
قابلة لمخاصمة
قابلة لتنفيذ
```

### 5.7 فلاتر العقود المنتهية

```text
منتهية بالتوقيع
منتهية بالتنفيذ
منتهية بالإلغاء
منتهية بانسحاب الأطراف
منتهية بانتهاء المدة
```

### 5.8 فلاتر الإجراءات المنتهية

```text
منجزة
مرفوضة
متوقفة
منتهية بتسليم النتيجة
منتهية لعدم متابعة
```

### 5.9 فلاتر الشركات المنتهية

```text
تأسيس منجز
تعديل منجز
ملف مؤرشف
شركة مشطوبة / منحلة إن وجدت
```

### 5.10 فلاتر الوكالات المنتهية

```text
منتهية
ملغاة
معزول عنها
اعتزال
مرتبطة بملفات منتهية فقط
```

## 6. بطاقة الملف المنتهي

### 6.1 أعلى البطاقة

```text
رقم ملف المكتب
نوع الملف
التصنيف الفرعي
سبب الإغلاق
```

### 6.2 العنوان

```text
عنوان الملف / الموضوع
```

### 6.3 الأطراف

حسب النوع:

```text
الموكل / الخصم
صاحب العلاقة / الجهة
الأطراف المتعاقدة
الشركة / الشركاء
الموكلون / المحامون
```

### 6.4 معلومات الإغلاق

```text
تاريخ الإغلاق
سبب الإغلاق
نتيجة الملف
هل الإغلاق الإداري مكتمل؟
```

مثال:

```text
سبب الإغلاق: شطب الدعوى
الإغلاق الإداري: غير مكتمل
إجراء لاحق متاح: تجديد الدعوى
```

### 6.5 المستندات والأصل الورقي

```text
عدد المستندات
هل يوجد أصل ورقي؟
مكان الأصل الورقي
هل تم تسليم مستندات؟
محضر تسليم موجود / غير موجود
```

### 6.6 المالية

```text
الرصيد
أتعاب متبقية
مصاريف غير مسددة
آخر سند قبض
```

لا تظهر كعمل يومي إلا كتنبيه أرشيفي أو مالي.

### 6.7 الإجراءات اللاحقة

تظهر فقط إذا تنطبق:

```text
تجديد الدعوى
استئناف
طعن بالنقض
مخاصمة
فتح تنفيذ
طباعة لا مانع
إعادة فتح كملف جارٍ
دعوى جديدة مرتبطة
```

مع مراعاة الصلاحية.

### 6.8 أزرار البطاقة المنتهية

الأزرار العامة:

```text
فتح الملف
طباعة ملخص
كشف مستندات
كشف مالي
```

أزرار خاصة:

```text
إجراءات لاحقة / طعون
طباعة لا مانع
محضر تسليم مستندات
إعادة فتح — للمالك / المدير فقط
```

لا تظهر الأزرار التالية على الملف المنتهي:

```text
إضافة جلسة عادية
إضافة أمر عمل عادي
موعد تشغيل يومي
```

إلا إذا تم إعادة فتح الملف أو فتح مرحلة لاحقة.

## 7. فتح الملف من الشاشتين

### 7.1 من الملفات الجارية

يفتح الملف في وضعه التشغيلي الكامل.

مثال دعوى جارية:

```text
دعوى Workspace
تبويب النواقص ظاهر
مواعيد وأوامر عمل كاملة
```

### 7.2 من الملفات المنتهية

يفتح نفس الـ Workspace لكن بوضع منتهي.

مثال دعوى منتهية:

```text
نفس الدعوى Workspace
قراءة فقط غالباً
بدون نواقص تشغيلية
مع إجراءات لاحقة / طعون
مع Checklist إغلاق إداري
```

```text
ثم آخر تحديث
```

```text

ثم التي لها إجراءات لاحقة

ثم آخر تحديث
```

## 9. حالات الفراغ

### 9.1 لا توجد ملفات جارية

النص:

```text
لا توجد ملفات جارية حالياً.
يمكن إنشاء ملف جديد من مكتب العمل > إنشاء جديد، أو إدخال أرشيف جارٍ من إدخال الأرشيف القديم.
```

الأزرار حسب الصلاحية:

```text
الانتقال إلى مكتب العمل
إدخال أرشيف قديم
```

### 9.2 لا توجد ملفات منتهية

النص:

```text
لا توجد ملفات منتهية محفوظة حالياً.
يمكن إدخال أرشيف منتهٍ من إدخال الأرشيف القديم.
```

الزر حسب الصلاحية:

```text
إدخال الأرشيف القديم
```

```text
إنشاء محضر تسليم مستندات
تعديل موقع أصل ورقي
إضافة سند قبض
إضافة مصروف
```

## 12. ممنوعات التصميم

لا يسمح بـ:

```text
إنشاء ملف جديد من شاشة الملفات الجارية
إنشاء ملف جديد من شاشة الملفات المنتهية
إدخال أرشيف من الملفات الجارية كزر إنشاء عشوائي
تحويل ملف منتهٍ إلى جارٍ بلا صلاحية وسبب
إظهار نواقص تشغيلية مزعجة على الأرشيف المنتهي
تكرار الوكالة كنسخة مستقلة داخل كل ملف بدل ربطها
فصل المستندات عن الملف في قسم جانبي مشتت
```

## 13. الخلاصة التنفيذية

تعتمد شاشة الملفات الجارية والمنتهية المنطق التالي:

```text
الملفات الجارية = لوحة تشغيل يومية.
الملفات المنتهية = أرشيف بحث وإجراءات لاحقة.
إدخال الأرشيف القديم = مسار مستقل.
إنشاء الجديد = مكتب العمل فقط.
كل بطاقة تفتح Workspace متخصص حسب نوع الملف.
النواقص التشغيلية للجاري فقط.
الإغلاق الإداري والأصل الورقي ومحاضر التسليم تظهر في المنتهي.
```

هذه الخطة مرجع تنفيذ شاشة ملفات المكتب بعد اعتمادها، ويجب ألا يتم تنفيذ أزرار أو مداخل تخالف الفصل بين الإنشاء الجديد والأرشيف القديم والملفات الجارية والمنتهية.
