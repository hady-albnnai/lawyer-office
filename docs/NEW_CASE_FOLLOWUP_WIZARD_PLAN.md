# خطة ويزارد إنشاء دعوى جديدة / دعوى متابعة

تاريخ الاعتماد: 2026-07-18

هذه الخطة تضبط مسار إنشاء الدعوى بعد نزول التطبيق من:

```text
مكتب العمل > إنشاء جديد > دعوى
```

ولا تخص إدخال الأرشيف القديم. الأرشيف القديم يبقى له مسار مستقل يبدأ من:

```text
ملفات المكتب > إدخال الأرشيف القديم
```

---

## 1. الهدف

إنشاء ملف دعوى جديد داخل المكتب بطريقة قانونية عملية، بحيث يظهر لاحقاً في:

```text
الملفات الجارية
مكتب العمل
التقويم
أوامر العمل
المالية
المستندات
سجل الأحداث
```

الويزارد يجب أن يميز من البداية بين:

```text
دعوى جديدة
دعوى متابعة
```

لأن بيانات الدعوى المتابعة تختلف عن الدعوى التي يبدأها المكتب من الصفر.

---

## 2. قاعدة المسارات

### 2.1 الدعوى بعد نزول التطبيق

تبدأ فقط من:

```text
مكتب العمل > إنشاء جديد > دعوى
```

ولا تبدأ من شاشة ملفات المكتب.

### 2.2 الأرشيف القديم

لا يستخدم هذا الويزارد كبداية مباشرة، بل يبدأ من:

```text
إدخال الأرشيف القديم
```

ثم يحدد:

```text
أرشيف جارٍ
أرشيف منتهٍ
```

ثم نوع الملف والتفاصيل.

### 2.3 النتيجة النهائية

رغم اختلاف مسارات البداية، يجب أن تنتهي كل المسارات إلى نفس شاشة الدعوى `Workspace`:

```text
دعوى جديدة
دعوى متابعة
أرشيف جارٍ
أرشيف منتهٍ
```

لكن بحالات وبيانات مختلفة.

## 3. الخطوة الأولى — نوع الإدخال

العنوان:

```text
ما نوع الدعوى التي تريد إدخالها؟
```

### 3.1 دعوى جديدة

الخيار:

```text
دعوى جديدة
```

الإيضاح:

```text
استخدم هذا الخيار عندما يبدأ المكتب الدعوى من بدايتها، ولم تكن منظورة سابقاً أمام المحكمة بواسطة محامٍ آخر.
```

أمثلة:

```text
الموكل حضر لأول مرة ويريد إقامة دعوى.
المكتب سيحرر الاستدعاء ويسجل الدعوى.
لا توجد جلسات أو إجراءات قضائية سابقة في هذه الدعوى.
```

أثر الاختيار:

```text
لا يطلب بيانات محامٍ سابق.
لا يطلب لا مانع.
لا يطلب جلسات سابقة.
يركز على التحضير والتسجيل والموعد القادم.
```

### 3.2 دعوى متابعة

الخيار:

```text
دعوى متابعة
```

الإيضاح:

```text
استخدم هذا الخيار عندما يستلم المكتب دعوى قائمة سابقاً أو منظورة أمام المحكمة، وكان يتابعها محامٍ آخر أو الموكل بنفسه.
```

أمثلة:

```text
الدعوى لها رقم أساس سابق.
يوجد جلسات سابقة.
صدر فيها قرار أو أكثر.
الموكل نقل الملف من محامٍ سابق.
المكتب دخل على الدعوى في مرحلة لاحقة.
```

أثر الاختيار:

```text
يطلب بيانات المحامي السابق إن وجد.
يطلب وثيقة لا مانع عند اللزوم.
يطلب الجلسات أو القرارات السابقة.
يطلب المستندات المستلمة من الملف السابق.
```

## 4. الخطوة الثانية — بيانات ملف المكتب

العنوان:

```text
بيانات ملف المكتب
```

الحقول:

```text
رقم ملف المكتب
تاريخ فتح الملف
المحامي المسؤول
المحامون المساعدون إن وجدوا
مصدر الملف
ملاحظات داخلية
```

### 4.1 رقم ملف المكتب

الإيضاح:

```text
هذا رقم داخلي خاص بالمكتب، وليس رقم أساس الدعوى في المحكمة.
```

أمثلة:

```text
2026/001
C-2026-015
```

قاعدة إلزامية:

```text
رقم ملف المكتب ≠ رقم أساس الدعوى
```

تنبيه واجهة:

```text
تنبيه: رقم ملف المكتب يستخدم للبحث الداخلي والأرشفة والمالية، أما رقم الأساس فيدخل لاحقاً ضمن بيانات المحكمة.
```

### 4.2 تاريخ فتح الملف

الإيضاح:

```text
تاريخ فتح الملف داخل المكتب، وليس بالضرورة تاريخ تسجيل الدعوى في المحكمة.
```

### 4.3 المحامي المسؤول

الإيضاح:

```text
المحامي المسؤول عن متابعة الملف داخل المكتب. يجب اختياره من مستخدمي التطبيق.
```

### 4.4 المحامون المساعدون

الإيضاح:

```text
محامون أو موظفون يساعدون في متابعة الملف، دون أن يكونوا المسؤولين الأساسيين.
```

### 4.5 مصدر الملف

الخيارات:

```text
موكل مباشر
إحالة من محامٍ
إحالة من عميل سابق
شركة / مؤسسة
مصدر آخر
```

الإيضاح:

```text
يساعد المكتب لاحقاً في معرفة مصدر الأعمال والتقارير الإدارية.
```

## 5. الخطوة الثالثة — الموكلون

العنوان:

```text
الموكلون
```

القاعدة:

```text
يمكن إضافة أكثر من موكل.
```

حقول كل موكل:

```text
نوع الموكل: شخص / جهة
الاسم
الصفة في الدعوى
رقم الهاتف
العنوان
الرقم الوطني أو السجل إن وجد
ملاحظات
```

### 5.1 الصفة في الدعوى

الخيارات بحسب السياق:

```text
مدعي
مدعى عليه
مستأنف
مستأنف عليه
طاعن
مطعون ضده
طالب تنفيذ
منفذ ضده
صفة أخرى
```

الإيضاح:

```text
الصفة القانونية للموكل ضمن هذه الدعوى أو المرحلة. قد تتغير الصفة في مرحلة الاستئناف أو النقض.
```

زر:

```text
إضافة موكل
```

إيضاح الزر:

```text
استخدمه إذا كان في الدعوى أكثر من موكل، مثل ورثة أو شركاء أو عدة مدعين.
```


## 7. الخطوة الخامسة — وكالات الموكلين

العنوان:

```text
وكالات الموكلين
```

الخيارات:

```text
إضافة وكالة الآن
لا توجد وكالة حالياً
سأضيفها لاحقاً
```

### 7.1 إضافة وكالة الآن

الإيضاح:

```text
استخدم هذا الخيار إذا كانت وكالة الموكل موجودة وتريد تسجيل بياناتها وإرفاق صورتها.
```

الحقول:

```text
نوع سند التوكيل
رقم سجل الوكالة
الرقم الأبيض
تاريخ ووقت تنظيم الوكالة
فرع النقابة
مندوب الوكالات
هاتف المندوب
الموكلون المشمولون بهذه الوكالة
صورة سند الوكالة
ملاحظات / نطاق الوكالة
```

أنواع سند التوكيل:

```text
سند توكيل عام
سند توكيل خاص
سند توكيل خاص شرعي
```

### 7.2 لا توجد وكالة حالياً

الإيضاح:

```text
استخدم هذا الخيار إذا لم يبرز الموكل وكالة بعد. سيظهر نقص تحذيري داخل الملف.
```

الأثر:

```text
يضاف نقص: لا توجد وكالة.
ولا يمنع حفظ الدعوى.
```

### 7.3 سأضيفها لاحقاً

الإيضاح:

```text
استخدم هذا الخيار إذا كانت الوكالة موجودة لكنك لا تريد إدخالها الآن.
```

الأثر:

```text
يضاف تنبيه تذكيري بإكمال بيانات الوكالة لاحقاً.
```

## 8. الخطوة السادسة — نوع الدعوى والاختصاص

العنوان:

```text
تصنيف الدعوى
```

الخيارات الرئيسية:

```text
مدنية
تجارية
جزائية
أحوال شخصية
قضاء إداري
```

### 8.1 مدنية

الإيضاح:

```text
للدعاوى المدنية بين الأشخاص أو الجهات، مثل المطالبات، التعويض، الملكية، الإيجار، إزالة الشيوع، الحقوق الشخصية والعينية.
```

### 8.2 تجارية

الإيضاح:

```text
للنزاعات التجارية بين التجار أو الشركات أو المتعلقة بالأعمال التجارية، مثل العقود التجارية، الأوراق التجارية، الشركات، الإفلاس عند وجوده.
```

### 8.3 جزائية

الإيضاح:

```text
للدعاوى والشكاوى ذات الطابع الجرمي، أمام النيابة أو التحقيق أو محاكم الجزاء أو الجنايات أو الأحداث.
```

### 8.4 أحوال شخصية

الإيضاح:

```text
للقضايا المتعلقة بالزواج والطلاق والنفقة والحضانة والإرث والوصية وما يتصل بالمحاكم الشرعية أو المذهبية أو الروحية.
```

### 8.5 قضاء إداري

الإيضاح:

```text
للمنازعات مع الإدارة أو القرارات الإدارية أو القضايا العمالية أو المسلكية أو الدعاوى أمام القضاء الإداري.
```

## 9. الخطوة السابعة — المرحلة / المحكمة

بعد اختيار التصنيف الرئيسي، تظهر المراحل المناسبة.

### 9.1 المدنية

الخيارات:

```text
صلح مدني
بداية مدنية
استئناف مدني
نقض مدني
مخاصمة
تنفيذ مدني
```

إيضاحات الخيارات:

```text
صلح مدني: للدعاوى التي تنظر أمام محكمة الصلح، غالباً حسب نوع النزاع أو قيمته أو الاختصاص المحدد قانوناً.
بداية مدنية: للدعاوى التي تنظر أمام محكمة البداية المدنية كدرجة أولى في المسائل الخارجة عن اختصاص الصلح أو المحددة قانوناً.
استئناف مدني: لمتابعة حكم صادر عن محكمة صلح أو بداية أمام محكمة الاستئناف المدنية.
نقض مدني: للطعن أمام محكمة النقض في الأحكام القابلة للنقض.
مخاصمة: لدعوى مخاصمة القضاة أو حالات المخاصمة وفق شروطها، وليست مرحلة عادية لكل دعوى.
تنفيذ مدني: لمتابعة تنفيذ حكم أو سند تنفيذي أمام دائرة التنفيذ المدني.
```

### 9.2 الجزائية

الخيارات:

```text
نيابة عامة
تحقيق
إحالة
أثناء المحاكمة
صلح جزاء
بداية جزاء
بداية جزاء معلوماتية
بداية جزاء مصرفية
بداية جزاء جمركية
استئناف جنح
جنايات
أحداث
نقض جزائي
مخاصمة
تنفيذ جزائي
```

إيضاحات الخيارات:

```text
نيابة عامة: عندما تكون الشكوى أو الملف في مرحلة النيابة العامة قبل التحقيق أو الإحالة.
تحقيق: عندما يكون الملف أمام قاضي التحقيق أو قاضي التحقيق المختص.
إحالة: عندما يكون الملف في مرحلة قاضي الإحالة أو قرار الإحالة للمحاكمة.
أثناء المحاكمة: عندما تكون الدعوى الجزائية منظورة فعلياً أمام المحكمة ولم تحدد مرحلة أدق بعد.
صلح جزاء: للدعاوى الجزائية الداخلة في اختصاص محكمة صلح الجزاء.
بداية جزاء: للدعاوى الجزائية الداخلة في اختصاص محكمة بداية الجزاء.
بداية جزاء معلوماتية: للدعاوى المتعلقة بالجرائم المعلوماتية أو التقنية أمام المحكمة المختصة.
بداية جزاء مصرفية: للدعاوى ذات الطابع المصرفي أو المالي أمام المحكمة الجزائية المختصة.
بداية جزاء جمركية: للدعاوى الجزائية الجمركية أو المتعلقة بالمخالفات والجرائم الجمركية.
استئناف جنح: لاستئناف الأحكام الجزائية القابلة للاستئناف.
جنايات: للدعاوى الجرمية الأشد التي تنظر أمام محكمة الجنايات.
أحداث: للدعاوى المتعلقة بالأحداث وفق المحكمة المختصة.
نقض جزائي: للطعن أمام محكمة النقض في الأحكام الجزائية القابلة للنقض.
مخاصمة: ليست طريقاً عادياً، وتستخدم عند توفر شروط المخاصمة.
تنفيذ جزائي: لمتابعة تنفيذ الحكم الجزائي أو ما يتصل به.
```

### 9.3 التجارية

الخيارات:

```text
بداية تجارية
استئناف تجاري
نقض تجاري
مخاصمة
```

إيضاحات الخيارات:

```text
بداية تجارية: للدعاوى التجارية كدرجة أولى أمام المحكمة المختصة.
استئناف تجاري: لاستئناف الحكم التجاري الصادر عن الدرجة الأولى.
نقض تجاري: للطعن بالنقض في الأحكام التجارية القابلة للنقض.
مخاصمة: تستخدم فقط عند توفر شروط المخاصمة، وليست مرحلة عادية.
```

### 9.4 الأحوال الشخصية

الخيارات:

```text
محكمة أحوال شخصية / شرعية / مذهبية / روحية
نقض أحوال شخصية
مخاصمة
```

إيضاحات الخيارات:

```text
محكمة أحوال شخصية / شرعية / مذهبية / روحية: للقضايا الأسرية والشرعية أو المذهبية أو الروحية حسب الاختصاص.
نقض أحوال شخصية: للطعن أمام محكمة النقض في الأحكام القابلة للنقض.
مخاصمة: تستخدم فقط عند توفر شروطها.
```

ملاحظة:

```text
لا نضع استئناف كخيار مستقل هنا بناءً على القرار المعتمد.
```

### 9.5 القضاء الإداري

الخيارات:

```text
محكمة عمالية
محكمة قضاء إداري
المحكمة الإدارية العليا
محكمة مسلكية
```

إيضاحات الخيارات:

```text
محكمة عمالية: للنزاعات العمالية عندما يكون مسارها ضمن المحكمة المختصة.
محكمة قضاء إداري: للدعاوى المتعلقة بالقرارات الإدارية أو المنازعات الإدارية.
المحكمة الإدارية العليا: للطعون أو الدعاوى الداخلة في اختصاص المحكمة الإدارية العليا.
محكمة مسلكية: للقضايا التأديبية أو المسلكية بحسب الاختصاص.
```

## 10. الخطوة الثامنة — تركيب اسم المحكمة

العنوان:

```text
بيانات المحكمة / المرجع القضائي
```

الحقول:

```text
النوع الرئيسي
النوع الفرعي
المحافظة
رقم الغرفة
اسم المحكمة الناتج
```

مثال:

```text
النوع الرئيسي: بداية
النوع الفرعي: مدنية
المحافظة: دمشق
رقم الغرفة: الثالثة
```

الناتج:

```text
البداية المدنية الثالثة في دمشق
```

الإيضاح:

```text
اسم المحكمة يبنى من عدة حقول حتى يكون البحث والترتيب والتقارير أدق، بدلاً من كتابته كنص واحد فقط.
```

خيار:

```text
تعديل الاسم يدوياً
```

إيضاحه:

```text
استخدمه إذا كان اسم المحكمة أو المرجع له صيغة خاصة غير موجودة في القوائم.
```

## 11. الخطوة التاسعة — رقم الأساس / المرجع القضائي

العنوان:

```text
بيانات القيد القضائي
```

الحقول:

```text
رقم الأساس
سنة الأساس
تاريخ التسجيل إن وجد
رقم القرار السابق إن وجد
رقم الطعن إن كانت مرحلة نقض
ملاحظات على القيد
```

الإيضاح:

```text
هذه أرقام المحكمة أو المرجع القضائي، وليست رقم ملف المكتب.
```

إذا كانت الدعوى جديدة ولم تسجل بعد، يظهر خيار:

```text
لم تسجل بعد
```

إيضاحه:

```text
استخدم هذا الخيار إذا كان الملف في مرحلة التحضير قبل تسجيل الدعوى في المحكمة.
```

الأثر:

```text
يضاف نقص تحذيري: لا يوجد رقم أساس.
ولا يمنع حفظ الملف.
```

```text
ملاحظات
```

### 12.3 الطلبات

الإيضاح:

```text
ما يطلبه الموكل من المحكمة أو الجهة القضائية.
```

---

## 13. الخطوة الحادية عشرة — الجلسات والموعد القادم

تختلف حسب نوع الإدخال.

### 13.1 في دعوى جديدة

الخيارات:

```text
لا يوجد موعد بعد
تحديد موعد مراجعة
تحديد موعد جلسة
```

الإيضاحات:

```text
لا يوجد موعد بعد: استخدمه إذا كان الملف في مرحلة التحضير ولا يوجد موعد معروف.
تحديد موعد مراجعة: استخدمه لتذكير المكتب بإجراء داخلي، مثل تحضير الاستدعاء أو مراجعة الموكل أو دفع رسم.
تحديد موعد جلسة: استخدمه إذا كانت الدعوى سجلت وأصبح لها موعد جلسة.
```

### 13.2 في دعوى متابعة

الحقول الإضافية:

```text
هل توجد جلسات سابقة؟
إضافة آخر جلسة
إضافة جلسات متعددة
هل صدر قرار سابق؟
إرفاق قرارات أو ضبوط سابقة
الموعد القادم
الإجراء القادم
```

الإيضاح:

```text
في دعوى المتابعة، المهم إدخال آخر وضع وصلت إليه الدعوى حتى يعرف المكتب من أين سيكمل.
```

---

## 14. الخطوة الخاصة بدعوى المتابعة — بيانات المحامي السابق

تظهر فقط إذا كان النوع:

```text
دعوى متابعة
```

الحقول:

```text
هل كان هناك محامٍ سابق؟
اسم المحامي السابق
رقم هاتفه
فرع النقابة إن وجد
هل يوجد لا مانع؟
إرفاق لا مانع
ملاحظات على استلام الملف
```

### 14.1 هل كان هناك محامٍ سابق؟

الخيارات:

```text
نعم
لا
غير معروف
```

الإيضاح:

```text
اختر نعم إذا كان الملف متابعاً سابقاً من محامٍ آخر.
```

### 14.2 لا مانع

الخيارات:

```text
مرفق
سيتم إحضاره لاحقاً
غير مطلوب حسب الحالة
```

الإيضاح:

```text
يستخدم لتوثيق انتقال الملف إلى المكتب عند الحاجة، ولا يمنع حفظ الملف إذا لم يكن مرفقاً، لكنه يظهر كنقص تحذيري.
```

## 15. الخطوة الثالثة عشرة — المستندات الأولية

العنوان:

```text
المستندات الأولية
```

أنواع مقترحة:

```text
وكالة
هوية / سجل
استدعاء
إيصال رسم
تبليغ
ضبط جلسة
حكم / قرار سابق
لا مانع
مراسلات
مستندات مؤيدة
أخرى
```

الإيضاح:

```text
أضف المستندات المتوفرة حالياً. يمكن استكمال المستندات لاحقاً من تبويب المستندات داخل ملف الدعوى.
```

## 16. الخطوة الرابعة عشرة — الأتعاب والمالية

العنوان:

```text
الأتعاب الاختيارية
```

الخيارات:

```text
إضافة اتفاق أتعاب الآن
تسجيل دفعة قبض
تأجيل المالية
```

### 16.1 إضافة اتفاق أتعاب الآن

الإيضاح:

```text
استخدمه لتسجيل الاتفاق المالي مع الموكل. هذا ليس سند قبض.
```

### 16.2 تسجيل دفعة قبض

الإيضاح:

```text
استخدمه إذا قبض المكتب مبلغاً فعلياً من الموكل. هذا ينشئ سند قبض.
```

### 16.3 تأجيل المالية

الإيضاح:

```text
استخدمه إذا لم يتم الاتفاق أو القبض الآن. يمكن الإضافة لاحقاً من تبويب المالية.
```

قاعدة واجهة:

```text
اتفاق الأتعاب ≠ سند قبض
```

## 17. الخطوة الخامسة عشرة — المراجعة والحفظ

العنوان:

```text
مراجعة الدعوى قبل الحفظ
```

يعرض:

```text
نوع الإدخال
رقم ملف المكتب
الموكلون
الخصوم
الوكالات
نوع الدعوى
المرحلة
المحكمة
رقم الأساس / لم تسجل بعد
الموعد القادم
المستندات
المالية
النواقص التحذيرية
```

### 17.1 النواقص التحذيرية

أمثلة:

```text
لا توجد وكالة
لا يوجد رقم أساس
لا يوجد موعد قادم
لا يوجد مستندات
لا يوجد لا مانع لدعوى متابعة
```

الإيضاح:

```text
النواقص لا تمنع الحفظ، لكنها تظهر داخل الملف لتذكير المكتب باستكمالها.
```

الأزرار:

```text
حفظ وفتح ملف الدعوى
حفظ وإضافة أمر عمل
رجوع للتعديل
إلغاء
```

## 18. بعد الحفظ

### 18.1 دعوى جديدة

تنشأ الدعوى بحالة مناسبة:

```text
قيد التحضير
```

أو:

```text
مسجلة
```

حسب وجود رقم أساس وموعد.

### 18.2 دعوى متابعة

تنشأ الدعوى بحالة مناسبة حسب البيانات:

```text
قيد المحاكمة
بانتظار قرار
معلقة
بانتظار تنفيذ
```

ويضاف إلى سجل الأحداث:

```text
تم فتح الملف كدعوى متابعة.
تم استلام الملف من محامٍ سابق: ...
تم إرفاق لا مانع إن وجد.
```

## 19. قواعد تنفيذية ملزمة

### 19.1 لا إنشاء من ملفات المكتب

هذا الويزارد لا يظهر في:

```text
الملفات الجارية
الملفات المنتهية
```

بل فقط من:

```text
مكتب العمل > إنشاء جديد
```

### 19.2 الأرشيف القديم له مسار مختلف

لا يخلط هذا الويزارد مع:

```text
إدخال الأرشيف القديم
```

لأن الأرشيف يبدأ بسؤال:

```text
أرشيف جارٍ
أرشيف منتهٍ
```

بينما الدعوى الجديدة تبدأ بسؤال:

```text
دعوى جديدة
دعوى متابعة
```

### 19.3 النواقص تحذيرية

النواقص التي تظهر عند الحفظ لا تمنع إنشاء الدعوى مبدئياً، لكنها تظهر داخل ملف الدعوى وفي كشوف النواقص.

### 19.4 القوائم يجب أن تبقى قابلة للتوسع

تصنيفات الدعاوى والمراحل والمحاكم والصفات والمستندات يجب ألا تكون مغلقة بشكل يمنع إضافة خيار مخصص عند الحاجة.

## 20. الخلاصة

يعتمد الويزارد المنطق التالي:

```text
هل الدعوى جديدة أم متابعة؟
من هم الموكلون؟
من هم الخصوم؟
هل توجد وكالة؟
ما نوع الدعوى؟
ما المرحلة والمحكمة؟
ما رقم الأساس أو هل لم تسجل بعد؟
ما موضوع الدعوى وطلباتها؟
هل توجد جلسات أو قرارات سابقة؟
هل يوجد محامٍ سابق ولا مانع؟
ما المستندات الأولية؟
هل توجد أتعاب أو قبض؟
ما النواقص قبل الحفظ؟
```

هذه الخطة مرجع تنفيذ ويزارد الدعوى الجديد بعد اعتمادها، ويجب عدم الخلط بينها وبين ويزارد إدخال الأرشيف القديم.
