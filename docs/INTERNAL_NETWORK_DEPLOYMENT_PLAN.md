# خطة التشغيل على شبكة داخلية

تاريخ الاعتماد: 2026-07-20

هذه الخطة تضبط طريقة تشغيل تطبيق ميزان على شبكة داخلية للمكاتب وشركات المحاماة، بما يسمح بتوسع الاستخدام من محامٍ فردي إلى مكتب كبير أو شركة محاماة.

---

## 1. القاعدة العامة

لا يجوز تشغيل نسخة متعددة المستخدمين عبر مشاركة ملف قاعدة بيانات SQLite على مجلد شبكة.

السبب:

```text
SQLite مناسب للاستخدام المحلي أو المستخدم الواحد.
فتح نفس ملف SQLite من عدة أجهزة عبر شبكة قد يؤدي إلى مشاكل قفل وفساد بيانات وضعف أداء.
```

لذلك تعتمد النسخة الشبكية على:

```text
سيرفر محلي داخل المكتب
قاعدة بيانات Client/Server
خدمة API محلية
تخزين مرفقات مركزي
```

---

## 2. نسخ التشغيل المقترحة

### 2.1 نسخة فردية / محلية

مناسبة للمحامي الفرد أو المكتب الصغير جداً.

```text
SQLite محلي
مرفقات محلية
نسخ احتياطي محلي
ترخيص جهاز واحد
لا تحتاج سيرفر داخلي
```

### 2.2 نسخة شبكة داخلية

مناسبة للمكاتب والشركات.

```text
جهاز رئيسي / سيرفر داخل المكتب
PostgreSQL أو قاعدة بيانات Client/Server
Mizan Local Server / API
مجلد مرفقات مركزي بإدارة السيرفر
أجهزة Client داخل الشبكة
ترخيص مرتبط بالسيرفر الداخلي
عدد مستخدمين محدد
```

---

## 3. البنية المقترحة لنسخة الشبكة

```text
أجهزة المستخدمين
        ↓
Mizan Windows Client
        ↓
Mizan Local Server داخل الشبكة
        ↓
PostgreSQL + Attachments Storage
```

---

## 4. لماذا Local API Server؟

بدلاً من أن يتصل كل تطبيق مباشرة بقاعدة البيانات، يفضّل وجود خدمة محلية وسيطة.

الفوائد:

```text
تطبيق الصلاحيات مركزياً
تطبيق الترخيص مركزياً
منع العبث المباشر بقاعدة البيانات
إدارة المرفقات بشكل موحد
تسجيل Audit موحد
إدارة النسخ الاحتياطي
تسهيل التحديثات لاحقاً
إمكانية إضافة API للموبايل أو الويب لاحقاً
```

---

## 5. قاعدة البيانات

الخيار المقترح للنسخة الشبكية:

```text
PostgreSQL
```

أسباب الاختيار:

```text
مناسبة لتعدد المستخدمين.
تدعم الكتابة المتزامنة.
مستقرة وقابلة للتوسع.
مناسبة لشبكات داخلية كبيرة.
تدعم النسخ الاحتياطي والاستعادة بشكل أفضل من ملف محلي مشترك.
```

---

## 6. المرفقات

في النسخة الشبكية، لا يرفع كل جهاز المرفقات مباشرة إلى مجلد مشترك بلا إدارة.

القاعدة:

```text
رفع المرفق يتم عبر Mizan Local Server.
السيرفر يحفظ الملف في مجلد مركزي.
السيرفر يسجل بيانات الملف في قاعدة البيانات.
```

هذا يمنع:

```text
ضياع الملفات
تكرار غير مضبوط
كسر روابط المستندات
اختلاف أسماء ومسارات الملفات بين الأجهزة
```

---

## 7. إعداد نسخة الشبكة عند العميل

خطوات التثبيت المقترحة:

```text
اختيار جهاز رئيسي أو سيرفر داخل المكتب.
تثبيت PostgreSQL.
تثبيت Mizan Local Server.
تحديد مجلد المرفقات المركزي.
إعداد النسخ الاحتياطي.
استيراد أو إنشاء الترخيص الشبكي.
تثبيت Mizan Client على باقي الأجهزة.
ربط الأجهزة بعنوان السيرفر المحلي.
```

مثال:

```text
http://192.168.1.10:PORT
```

---

## 8. الترخيص في نسخة الشبكة

الترخيص يرتبط بـ:

```text
جهاز السيرفر الداخلي
```

وليس بكل جهاز Client.

يحدد الترخيص:

```text
اسم العميل
نوع النسخة: شبكة
عدد المستخدمين المسموح
مدة الترخيص
الميزات المفعلة
بصمة السيرفر الداخلي
```

---

## 9. عدد المستخدمين والجلسات

Mizan Local Server يجب أن يتحقق من:

```text
عدد المستخدمين المسموح في الترخيص
عدد الجلسات الحالية
عدد المستخدمين النشطين
```

إذا تم تجاوز العدد:

```text
تم تجاوز عدد المستخدمين المسموح به في الترخيص.
```

---

## 10. الصلاحيات في الشبكة

الصلاحيات تدار مركزياً في قاعدة الشبكة.

```text
مالك المكتب
مدير المكتب
محامي أستاذ
```

والقوائم المرجعية مثل المحامين المتمرنين والمعقبين تبقى مركزية لكل الأجهزة.

---

## 11. النسخ الاحتياطي في الشبكة

يجب دعم نسخ احتياطي مركزي يشمل:

```text
قاعدة البيانات
المرفقات
ملف الترخيص
إعدادات السيرفر
```

أولويات النسخ الاحتياطي:

```text
نسخ يومي تلقائي
نسخ يدوي عند الحاجة
حفظ نسخة على قرص خارجي أو مسار شبكة آخر
سجل عمليات النسخ والاستعادة
```

---

## 12. الاستعادة

استعادة نسخة الشبكة عملية حساسة، ويجب أن تكون للمالك فقط.

عند الاستعادة:

```text
إيقاف الخدمة مؤقتاً
استعادة قاعدة البيانات
استعادة المرفقات
فحص الترخيص
إعادة تشغيل الخدمة
تسجيل الحدث
```

---

## 13. التوسع

عند كبر المكتب يمكن تحسين:

```text
جهاز السيرفر
أداء PostgreSQL
مساحة التخزين
سرعة الشبكة
سياسة النسخ الاحتياطي
عدد المستخدمين في الترخيص
```

---

## 14. العلاقة مع النسخة المحلية

يجب أن يبقى التطبيق قادراً على العمل كنسخة فردية.

الفرق:

```text
النسخة الفردية: SQLite محلي.
النسخة الشبكية: Client يتصل بـ Local Server.
```

لا يجب خلط ملف SQLite محلي مع قاعدة الشبكة.

---

## 15. ترحيل بيانات من نسخة فردية إلى شبكة

عند ترقية عميل من نسخة فردية إلى نسخة شبكة، يجب دعم مسار ترحيل لاحقاً:

```text
تصدير بيانات النسخة المحلية
استيرادها إلى قاعدة الشبكة
نقل المرفقات
إعادة ربط الملفات
التحقق من الترقيم والبيانات
```

هذه ميزة مهمة لكن يمكن تنفيذها بعد النسخة الأولى للشبكة.

---

## 16. الخلاصة التنفيذية

القواعد المعتمدة:

```text
لا تستخدم SQLite عبر مجلد شبكة مشترك.
النسخة الفردية تستخدم SQLite محلياً.
النسخة الشبكية تستخدم Local Server + PostgreSQL + مرفقات مركزية.
العملاء يتصلون بالسيرفر المحلي عبر الشبكة الداخلية.
الترخيص الشبكي يرتبط بالسيرفر الداخلي وعدد المستخدمين.
النسخ الاحتياطي والاستعادة مركزية وحساسة.
التصميم يجب أن يسمح لاحقاً بالتوسع وربما بإضافة موبايل أو ويب عبر نفس API.
```
