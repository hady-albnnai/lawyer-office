# خطة ملف الإجراء / المعاملة

تاريخ الاعتماد: 2026-07-20

هذه الخطة تضبط ملف `إجراء / معاملة` داخل التطبيق وفق القاعدة المعتمدة:

```text
الإجراء / المعاملة ملف مستقل وليس مهمة.
التطبيق لا يعيد إدخال كل تفاصيل الإجراء داخل حقول تفصيلية مرهقة.
تفاصيل الإجراء الكاملة تبقى في المستندات والمرفقات.
التطبيق يحفظ بيانات الغلاف والفهرسة والبحث والربط والمتابعة.
```

---

## 1. فلسفة ملف الإجراء

ملف الإجراء في التطبيق هو:

```text
غلاف إداري / قانوني للإجراء
فهرسة وبحث
جهة ومكان
أطراف / أصحاب علاقة
مستندات ومرفقات
ربط مع دعوى أو عقد أو شركة أو وكالة أو إجراء آخر
مواعيد ومراجعات عند الحاجة
مالية ومصاريف
سجل أحداث
```

وليس شاشة لإعادة كتابة كل تفاصيل المعاملة الرسمية داخل التطبيق.

تفاصيل الإجراء الدقيقة تبقى داخل:

```text
طلب الإجراء
الاستدعاء
الوثيقة الرسمية
الكتاب المرفق
الإيصال
صورة المعاملة
المرفقات
```

---

## 2. رقم ملف الإجراء

رقم ملف المكتب للإجراء يولد تلقائياً حسب قاعدة الترقيم:

```text
إجراء/2026/0001
```

ولا يعدل بعد الحفظ.

---

## 3. بداية ويزارد الإجراء

يبدأ الإجراء الجديد من:

```text
مكتب العمل > إنشاء جديد > إجراء / معاملة
```

أما الأرشيف القديم للإجراءات فيبدأ من:

```text
ملفات المكتب > إدخال الأرشيف القديم
```

---

## 4. نوع ملف الإجراء

الخيارات:

```text
إجراء / معاملة جديدة
إجراء / معاملة متابعة
```

### 4.1 إجراء / معاملة جديدة

الإيضاح:

```text
استخدم هذا الخيار عندما يبدأ المكتب متابعة إجراء أو معاملة من بدايتها بعد نزول التطبيق.
```

### 4.2 إجراء / معاملة متابعة

الإيضاح:

```text
استخدم هذا الخيار عندما يستلم المكتب معاملة كانت قائمة سابقاً أو بدأها الموكل أو شخص آخر، والمكتب سيكمل المتابعة.
```

## 5. التصنيف الرئيسي والفرعي

يعتمد الإجراء على:

```text
تصنيف رئيسي
تصنيف فرعي
```

ويجب أن تتضمن القوائم الافتراضية كل ما يمكن استخلاصه من الواقع والقانون السوري قدر الإمكان، مع إفساح المجال لإضافة تصنيفات جديدة من المستخدم.

### 5.1 التصنيفات الرئيسية المقترحة

```text

معاملة مالية / ضريبية
معاملة كاتب عدل

معاملة مركبات / نقل
إجراء قضائي غير دعوى
إجراء آخر
```

## 7. الجهة والهيكل الإداري والمكاني

الجهة يجب أن تكون مرنة وبسيطة، مع حقول تسمح بوصفها إدارياً ومكانياً دون إرهاق المستخدم.

### 7.1 بيانات الجهة

```text

التصنيف الإداري
التصنيف المكاني

اسم الموظف أو المرجع إن وجد

```

## 9. المستند / الإجراء المرفق

كل إجراء يجب أن يتيح إرفاق الإجراء الذي تم أو الوثيقة الناتجة أو الطلب أو المستندات المرتبطة به.

أنواع مرفقات مقترحة:

```text
طلب / استدعاء
كتاب رسمي
إيصال رسم
براءة ذمة
بيان قيد
قرار / موافقة
رفض / إشعار نقص
صورة معاملة
وثيقة صادرة
وثيقة مطلوبة
أخرى
```

القاعدة:

```text
تفاصيل الإجراء تبقى في المستندات والمرفقات.
التطبيق يحفظ الغلاف والربط والبحث والمتابعة.
```

## 10. الخطوات والمراجعات

يمكن إضافة خطوات أو مراجعات عند الحاجة، لكنها ليست إلزامية لكل إجراء.

كل خطوة:

```text
اسم الخطوة
الجهة / القسم
تاريخ المراجعة
من قام بالمراجعة
ما تم
النقص المطلوب
الإجراء القادم
الموعد القادم
مرفقات
ملاحظات
```

أمثلة:

```text
تقديم الطلب
دفع الرسم
تدقيق المستندات
استكمال نقص
مراجعة المدير
استلام كتاب
تسجيل وارد
استلام النتيجة
```

يمكن لاحقاً توفير خطوات جاهزة حسب نوع الإجراء، لكن يجب أن يبقى المستخدم قادراً على إضافة خطوة مخصصة.

## 11. ربط الإجراء بالملفات الأخرى

الإجراء يجب دائماً أن يدعم:

```text
ربطه بدعوى قائمة
إنشاء دعوى جديدة يكون الإجراء أساسها
ربطه بعقد
ربطه بشركة
ربطه بوكالة
ربطه بإجراء آخر
```

### 11.1 ربط بدعوى قائمة

يختار المستخدم دعوى موجودة ويحدد نوع العلاقة.

أمثلة علاقة:

```text
إثبات صفة
مستند مؤيد للدعوى
إجراء ممهد للدعوى
إجراء مطلوب أثناء الدعوى
إجراء لاحق للدعوى
مرتبط بنفس الموضوع
```

### 11.2 إنشاء دعوى يكون الإجراء أساسها

يظهر خيار:

```text
فتح دعوى مرتبطة
```

ويجب أن يضاف هذا الخيار إلى ويزارد إنشاء الدعوى.

عند استخدامه:

```text
ينتقل المستخدم إلى ويزارد إنشاء الدعوى.
تظهر الدعوى كدعوى مرتبطة بالإجراء.
يمكن نقل الأطراف المختارة.
يمكن ربط مستندات مختارة.
يسجل في الدعوى أن أساس فتحها هو هذا الإجراء.
```

أمثلة:

```text
رفض تظلم إداري → دعوى قضاء إداري
حصر إرث → دعوى إزالة شيوع أو قسمة أو تثبيت صفة
رفض معاملة عقارية → دعوى مدنية / إدارية حسب الحالة
```

## 17. النواقص التحذيرية

أمثلة:

```text
لا توجد جهة محددة
لا يوجد تصنيف فرعي
لا يوجد مستند مرفق
لا يوجد رقم معاملة رغم الحاجة
لا توجد نتيجة مرفقة رغم إغلاق الإجراء كمنجز
أصل ورقي غير محدد
```

النواقص لا تمنع الحفظ مبدئياً، لكنها تظهر في الملفات الجارية وكشوف النواقص.

## 18. الخلاصة التنفيذية

القواعد المعتمدة:

```text
الإجراء / المعاملة ملف مستقل وليس مهمة.
تفاصيل الإجراء تبقى في المستندات والمرفقات.
التطبيق يحفظ الغلاف والفهرسة والربط والمتابعة.
التصنيف رئيسي ثم فرعي، مع قوائم سورية واسعة وقابلة للإضافة.
الجهة لها اسم وتصنيف إداري وتصنيف مكاني.
الإجراء يمكن دائماً ربطه بدعوى قائمة أو إنشاء دعوى يكون هو أساسها.
خيار إنشاء دعوى مرتبطة يجب أن يدخل ضمن ويزارد إنشاء الدعوى.
```
