# قواعد ترقيم ملفات المكتب وربط الإجراءات بالملفات

تاريخ الاعتماد: 2026-07-20

هذه الوثيقة تثبت قاعدتين أساسيتين في هيكل مكتب المحامي السوري:

```text
1. رقم ملف المكتب يولد تلقائياً حسب نوع الملف وببادئات عربية.
2. الإجراء / المعاملة يبقى ملفاً مستقلاً، ويمكن ربطه بدعوى أو عقد أو شركة أو وكالة أو إجراء آخر بعلاقة واضحة دون دمج الملفات.
```

---

## 1. رقم ملف المكتب

### 1.1 القاعدة العامة

رقم ملف المكتب هو رقم داخلي للتطبيق والمكتب، ويستخدم في:

```text
البحث الداخلي
بطاقات الملفات
التقارير
الطباعة
ربط المستندات
ربط المالية
ربط أوامر العمل
الأرشيف الداخلي
```

ولا يجوز خلطه مع:

```text
رقم الأساس القضائي
رقم المعاملة لدى الجهة
رقم الوكالة
رقم العقد
رقم السجل التجاري
رقم الملف الورقي القديم
```

---

## 2. التوليد التلقائي حسب نوع الملف

يعتمد التطبيق توليد رقم ملف المكتب تلقائياً حسب نوع الملف.

كل نوع ملف له تسلسل مستقل:

```text
الدعاوى لها تسلسل خاص.
الإجراءات / المعاملات لها تسلسل خاص.
العقود لها تسلسل خاص.
الشركات لها تسلسل خاص.
الوكالات لها تسلسل خاص.
```

الصيغة المعتمدة:

```text
دعوى/2026/0001
إجراء/2026/0001
عقد/2026/0001
شركة/2026/0001
وكالة/2026/0001
```

يمكن أن يوجد في نفس السنة:

```text
دعوى/2026/0001
إجراء/2026/0001
عقد/2026/0001
شركة/2026/0001
وكالة/2026/0001
```

لأن كل نوع له بادئة عربية مختلفة وتسلسل مستقل.

لكن لا يجوز أن يوجد ملفان من نفس النوع بنفس الرقم:

```text
دعوى/2026/0001
دعوى/2026/0001
```

هذا ممنوع تماماً.

---

## 3. البادئات العربية

تعتمد البادئات العربية حتى يكون الرقم مفهوماً للمستخدم مباشرة.

البادئات الأساسية:

```text
دعوى
إجراء
عقد
شركة
وكالة
```

ولا تعتمد البادئات الأجنبية مثل:

```text
C
P
K
CO
POA
```

في الواجهة الموجهة للمستخدم.

---

## 4. وقت توليد الرقم

يتولد رقم ملف المكتب عند:

```text
الحفظ النهائي للملف
```

ولا يتولد بمجرد فتح الويزارد.

أثناء الويزارد تظهر عبارة:

```text
سيتم توليد رقم ملف المكتب تلقائياً عند الحفظ.
```

ويمكن عرض رقم متوقع مثل:

```text
الرقم المتوقع: دعوى/2026/0008
```

مع تنبيه:

```text
قد يتغير الرقم عند الحفظ إذا حفظ مستخدم آخر ملفاً من نفس النوع قبلك.
```

---

## 5. منع التكرار

يجب أن يضمن التطبيق تقنياً:

```text
كل نوع ملف له عداد مستقل.
التوليد يتم أثناء الحفظ النهائي.
لا يصدر رقمان متشابهان لنفس النوع.
إذا حفظ مستخدمان بنفس اللحظة، يأخذ الأول الرقم التالي، والثاني الرقم الذي بعده.
```

التفرد المطلوب يكون على مستوى:

```text
نوع الملف + رقم ملف المكتب
```

وبغض النظر عن حالة الملف:

```text
جارٍ
منتهٍ
أرشيفي
ملغى
```

أي لا يجوز تكرار رقم دعوى في دعوى أخرى حتى لو كانت الأولى منتهية أو ملغاة.

---

## 6. عدم تعديل رقم ملف المكتب بعد الحفظ

### 6.1 القرار المعتمد

رقم ملف المكتب:

```text
غير قابل للتعديل بعد الحفظ حتى للمدير.
```

لا توجد صلاحية لتعديل الرقم الداخلي بعد إنشائه.

### 6.2 سبب القرار

عدم تعديل الرقم يحمي:

```text
الأرشيف
الطباعة
الإحالات الداخلية
البحث
المالية
المستندات
أوامر العمل
سجل الأحداث
```

ويمنع التلاعب أو الالتباس.

### 6.3 معالجة الخطأ

إذا حدث خطأ قبل الحفظ:

```text
يلغي المستخدم الويزارد أو يرجع لتعديل نوع الملف.
ولا يكون قد تولد رقم نهائي.
```

إذا حدث الخطأ بعد الحفظ:

```text
لا يعدل الرقم.
يتم إلغاء / إبطال الملف بسبب خطأ إدخال.
يفتح ملف صحيح جديد.
يربط الملف الملغى بالملف الصحيح كملاحظة إدارية عند الحاجة.
```

مثال:

```text
إجراء/2026/0004 = ملغى بسبب خطأ في نوع الملف.
دعوى/2026/0007 = الملف الصحيح الجديد.
```

ويظهر في سجل الأحداث:

```text
تم إلغاء الملف إجراء/2026/0004 بسبب خطأ في نوع الملف.
تم فتح الملف الصحيح دعوى/2026/0007 وربطه بالملف الملغى.
```

---

## 7. الأرشيف القديم والرقم الورقي القديم

الأرشيف القديم قد يحتوي رقماً قديماً مثل:

```text
2018/45
ملف 12
دفتر 3 / رقم 18
```

هذا الرقم لا يصبح تلقائياً رقم ملف المكتب الجديد.

يعتمد حقل مستقل باسم:

```text
رقم الملف القديم / الورقي
```

أما رقم ملف المكتب في التطبيق فيتولد تلقائياً حسب نوع الملف، مثل:

```text
دعوى/2026/0045
```

وتظهر البطاقة مثلاً:

```text
رقم ملف المكتب: دعوى/2026/0045
رقم الملف القديم: 2018/45
```

البحث يجب أن يبحث في:

```text
رقم ملف المكتب
رقم الملف القديم / الورقي
رقم الأساس
رقم المعاملة
رقم الوكالة
رقم العقد
```

لكن الرقم الأساسي في التطبيق يبقى:

```text
رقم ملف المكتب
```

---

## 8. الروابط الداخلية

رغم أهمية رقم ملف المكتب في الواجهة والطباعة، يجب أن تعتمد الروابط الداخلية تقنياً على:

```text
id داخلي ثابت
```

وليس على رقم ملف المكتب.

هذا يضمن أن:

```text
المستندات لا تضيع.
المالية لا تضيع.
أوامر العمل لا تضيع.
الملفات المرتبطة لا تضيع.
سجل الأحداث لا يختلط.
```

---

## 9. ربط الإجراء / المعاملة بالملفات الأخرى

### 9.1 القاعدة العامة

الإجراء / المعاملة:

```text
ملف مستقل وليس مهمة.
```

وقد يرتبط بـ:

```text
دعوى
عقد
شركة
وكالة
إجراء آخر
```

الربط لا يعني الدمج.

أي:

```text
الإجراء يبقى ملفاً مستقلاً وله رقم ملف مكتب خاص.
الدعوى أو العقد أو الشركة أو الوكالة تبقى ملفاً مستقلاً لها رقمها الخاص.
العلاقة بينهما تسجل كرابط مسمى يشرح سبب الربط.
```

مثال:

```text
إجراء/2026/0003 — حصر إرث
مرتبط بـ:
دعوى/2026/0011 — إزالة شيوع
العلاقة: إثبات صفة الورثة
```

---

## 10. متى يكون الإجراء ملفاً مستقلاً؟

يكون الإجراء ملفاً مستقلاً إذا كان له:

```text
جهة مستقلة
رقم معاملة
مراجعات
مستندات
نتيجة
مالية أو رسوم
متابعة قد تطول
وقد يخدم أكثر من دعوى أو ملف
```

أمثلة:

```text
حصر إرث
معاملة فراغ عقاري
بيان قيد عقاري
اعتراض ضريبي
معاملة سجل تجاري
تصديق وكالة أو وثيقة
```

أما الأعمال الصغيرة داخل دعوى، مثل:

```text
مراجعة الديوان لتصوير ضبط
إبراز مذكرة
دفع رسم دعوى
استلام تبليغ
```

فهي غالباً:

```text
أمر عمل أو خطوة داخل الدعوى
```

وليست ملف إجراء مستقل.

---

## 11. أنواع علاقة الإجراء بالدعوى

عند ربط إجراء بدعوى، يسأل النظام:

```text
ما نوع علاقة هذا الإجراء بالدعوى؟
```

الخيارات العامة:

```text
إجراء ممهد للدعوى
مستند / نتيجة مؤيدة للدعوى
إجراء مطلوب أثناء الدعوى
إجراء لاحق للدعوى
الدعوى نشأت بسبب هذا الإجراء
إجراء بديل / مرتبط بنفس الموضوع
```

### 11.1 إجراء ممهد للدعوى

الإيضاح:

```text
استخدمه إذا كان هذا الإجراء تم قبل الدعوى أو كان لازماً لتحضيرها أو إثبات الصفة أو الحق.
```

أمثلة:

```text
حصر إرث قبل دعوى إزالة شيوع.
بيان قيد عقاري قبل دعوى تثبيت بيع.
كشف مالي قبل دعوى مطالبة.
وثيقة شركة قبل دعوى تجارية.
```

### 11.2 مستند / نتيجة مؤيدة للدعوى

الإيضاح:

```text
استخدمه إذا كانت نتيجة الإجراء ستستخدم كمستند داخل الدعوى، دون أن تكون الدعوى ناشئة عنه مباشرة.
```

أمثلة:

```text
حصر إرث لإثبات صفة الورثة.
بيان قيد عقاري لإثبات ملكية.
سجل تجاري لإثبات صفة شركة.
```

### 11.3 إجراء مطلوب أثناء الدعوى

الإيضاح:

```text
استخدمه إذا طلبت المحكمة أو احتاج المكتب إجراءً خارجياً أثناء سير الدعوى.
```

أمثلة:

```text
استخراج قيد عقاري أثناء دعوى.
تصديق وثيقة.
إحضار بيان عائلي.
استخراج سجل تجاري.
```

### 11.4 إجراء لاحق للدعوى

الإيضاح:

```text
استخدمه إذا نشأ الإجراء بعد صدور حكم أو قرار أو نتيجة في الدعوى.
```

أمثلة:

```text
تنفيذ نقل ملكية بعد حكم.
تسجيل حكم في السجل.
تصحيح قيد بناءً على حكم.
```

### 11.5 الدعوى نشأت بسبب هذا الإجراء

الإيضاح:

```text
استخدمه إذا انتهى الإجراء بنتيجة أو رفض أدى إلى فتح دعوى.
```

أمثلة:

```text
رفض تظلم إداري ثم فتح دعوى قضاء إداري.
رفض معاملة عقارية ثم إقامة دعوى.
رفض تسجيل شركة أو تعديل ثم إقامة دعوى.
```

### 11.6 إجراء بديل / مرتبط بنفس الموضوع

الإيضاح:

```text
استخدمه إذا كان الإجراء والدعوى مرتبطين بنفس الأطراف أو الموضوع، لكن لا يوجد سبب مباشر بينهما.
```

---

## 12. مثال حصر الإرث

عند وجود:

```text
إجراء/2026/0003 — حصر إرث للمرحوم أحمد
```

وربطه بدعوى، تظهر الخيارات المناسبة مثل:

```text
إثبات صفة الورثة في الدعوى
إجراء ممهد للدعوى
مستند مؤيد للدعوى
مستند مطلوب من المحكمة
إجراء مرتبط بنفس التركة / العقار
سبب فتح الدعوى
```

### 12.1 حصر إرث مع دعوى إزالة شيوع

العلاقة:

```text
إثبات صفة الورثة في الدعوى
```

أو:

```text
إجراء ممهد للدعوى
```

الأثر:

```text
يظهر حصر الإرث داخل الدعوى في قسم الملفات المرتبطة.
تظهر صورة حصر الإرث ضمن مستندات الدعوى كمستند مرتبط عند اختيارها.
لا ينسخ كملف جديد.
يبقى ملف حصر الإرث مستقلاً وله رقمه.
```

### 12.2 حصر إرث مع دعوى تثبيت بيع

العلاقة:

```text
مستند مؤيد للدعوى
```

أو:

```text
إثبات صفة أحد الأطراف
```

### 12.3 حصر إرث استخرج أثناء دعوى

العلاقة:

```text
إجراء مطلوب أثناء الدعوى
```

### 12.4 حصر إرث أدى لاحقاً لدعوى

العلاقة:

```text
الدعوى نشأت بسبب هذا الإجراء
```

أو:

```text
إجراء ممهد للدعوى
```

وفي هذه الحالة يمكن من ملف الإجراء الضغط على:

```text
فتح دعوى مرتبطة
```

---

## 13. شاشة ربط الإجراء بملف

عند الضغط على:

```text
ربط بملف
```

تظهر الخيارات:

```text
ربط بدعوى موجودة
إنشاء دعوى مرتبطة جديدة
ربط بعقد
ربط بشركة
ربط بوكالة
ربط بإجراء آخر
```

بعد اختيار الملف، تظهر حقول الربط:

```text
الملف المراد ربطه
نوع العلاقة
سبب الربط / ملاحظات
هل تريد إظهار مستندات الإجراء داخل الملف المرتبط؟
اختيار المستندات التي ستظهر
هل تريد إنشاء أمر عمل متعلق بهذا الربط؟
```

---

## 14. المستندات عند الربط

عند ربط إجراء بدعوى أو ملف آخر، لا تنسخ كل مستندات الإجراء تلقائياً.

يسأل النظام:

```text
ما المستندات التي تريد إظهارها داخل الملف المرتبط؟
```

مثال حصر إرث:

```text
صورة حصر الإرث
طلب حصر الإرث
إيصال الرسم
بيان عائلي
```

قد يختار المستخدم فقط:

```text
صورة حصر الإرث
```

فتظهر في مستندات الدعوى كـ:

```text
مستند مرتبط من ملف إجراء إجراء/2026/0003
```

وليس نسخة منفصلة بلا علاقة.

---

## 15. أثر الربط على الدعوى

داخل الدعوى يظهر قسم:

```text
ملفات مرتبطة
```

أو داخل الملخص:

```text
إجراءات مرتبطة
```

ويعرض مثلاً:

```text
حصر إرث — إثبات صفة الورثة
رقم ملف المكتب: إجراء/2026/0003
الحالة: منجز
النتيجة: تم استخراج حصر الإرث
المستندات المرتبطة: 1
```

وفي تبويب المستندات يظهر:

```text
حصر إرث — مرتبط من ملف إجراء إجراء/2026/0003
```

---

## 16. أثر الربط على الإجراء

داخل ملف الإجراء يظهر قسم:

```text
ملفات مرتبطة
```

مثال:

```text
دعوى إزالة شيوع — دعوى/2026/0011
نوع العلاقة: إثبات صفة الورثة
```

---

## 17. هل الربط يغير حالة الإجراء؟

ليس دائماً.

### 17.1 إذا كان الإجراء منجزاً

مثال:

```text
حصر إرث مستخرج
```

يبقى:

```text
منجزاً / منتهياً
```

ويرتبط بالدعوى كمستند أو إجراء سابق.

### 17.2 إذا كان الإجراء جارياً

يبقى جارياً ويظهر في:

```text
الملفات الجارية
مكتب العمل إذا له موعد
```

### 17.3 إذا كان الإجراء سبب فتح دعوى

عند فتح الدعوى من الإجراء:

```text
يمكن أن يصبح الإجراء منتهياً بسبب الرفض أو انتهاء المسار الإداري.
وتنشأ دعوى مرتبطة جديدة.
```

---

## 18. الربط مع باقي أنواع الملفات

### 18.1 إجراء ↔ عقد

مثال:

```text
عقد بيع عقار
معاملة فراغ عقاري
```

أنواع العلاقة:

```text
إجراء لتنفيذ العقد
إجراء ممهد للعقد
مستند مؤيد للعقد
إجراء لاحق للعقد
```

### 18.2 إجراء ↔ شركة

مثال:

```text
شركة
معاملة سجل تجاري
تعديل غاية
تغيير مدير
رفع رأس مال
```

أنواع العلاقة:

```text
إجراء تأسيس الشركة
إجراء تعديل لاحق
إجراء تصحيح بيانات
إجراء مرتبط بالسجل التجاري
```

### 18.3 إجراء ↔ وكالة

مثال:

```text
وكالة
تصديق صورة وكالة
استخراج صورة عن وكالة
تسجيل عزل / اعتزال
```

أنواع العلاقة:

```text
إجراء متعلق بالوكالة
إجراء استخراج صورة
إجراء تصديق
إجراء عزل / اعتزال
```

### 18.4 إجراء ↔ إجراء آخر

مثال:

```text
حصر إرث
ثم معاملة انتقال ملكية
ثم معاملة إفراز
```

أنواع العلاقة:

```text
إجراء سابق ممهد
إجراء لاحق
إجراء مكمل
إجراء بديل
```

---

## 19. فتح دعوى من إجراء

إذا انتهى الإجراء إلى رفض أو نزاع أو حاجة للجوء للقضاء، يظهر زر:

```text
فتح دعوى مرتبطة
```

مثال:

```text
إجراء: تظلم إداري
النتيجة: رفض التظلم
```

عند الضغط على `فتح دعوى مرتبطة` يسأل النظام:

```text
ما نوع الدعوى؟
هل تريد نقل الأطراف؟
هل تريد نقل المستندات؟
هل تريد نقل ملخص الوقائع؟
هل تريد ربط النتيجة كسبب فتح الدعوى؟
```

بعد الحفظ:

```text
ينشئ دعوى جديدة برقم ملف مكتب تلقائي مثل دعوى/2026/0011.
يربطها بالإجراء.
يظهر في الدعوى أن سببها رفض الإجراء.
يظهر في الإجراء أن دعوى مرتبطة فتحت عنه.
```

## 20. الخلاصة التنفيذية

القواعد المعتمدة:

```text
رقم ملف المكتب يولد تلقائياً حسب نوع الملف.
البادئة عربية حتى يفهمها المستخدم.
كل نوع ملف له تسلسل مستقل.
الرقم يولد عند الحفظ النهائي فقط.
رقم ملف المكتب غير قابل للتعديل بعد الحفظ حتى للمدير.
الأخطاء تعالج بإلغاء الملف وفتح ملف صحيح جديد، لا بتعديل الرقم.
الأرشيف القديم يحتفظ برقمه القديم في حقل مستقل.
الإجراء ملف مستقل وليس مهمة.
الإجراء يمكن ربطه بدعوى أو عقد أو شركة أو وكالة أو إجراء آخر.
الربط له نوع علاقة واضح.
الربط لا يدمج الملفات.
يمكن إظهار مستندات مختارة من الإجراء داخل الدعوى أو الملف المرتبط كمستندات مرتبطة.
يمكن فتح دعوى مرتبطة من إجراء عند الرفض أو النزاع.
```

هذه القواعد يجب تطبيقها في ويزاردات الإنشاء والأرشيف وشاشات الملفات الجارية والمنتهية وكل Workspace متخصص.
