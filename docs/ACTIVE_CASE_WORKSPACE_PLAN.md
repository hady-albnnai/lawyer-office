# خطة شاشة الدعوى الجارية — Workspace الدعوى

تاريخ الخطة: 2026-07-17

هذه الخطة تضبط طريقة عرض وإدارة الدعوى الجارية داخل ملفات المكتب، بعد الاتفاق على أن `الملفات الجارية` هي شاشة تعامل وبحث وتعديل، وأن إنشاء الملفات الجديدة يتم فقط من `مكتب العمل > إنشاء جديد`.

---

## 1. قاعدة رقم الملف ورقم الأساس

يجب الفصل دائماً بين:

```text
رقم ملف المكتب
رقم الأساس القضائي
```

- `رقم ملف المكتب`: رقم داخلي لإدارة الملف داخل المكتب والتطبيق.
- `رقم الأساس`: رقم قضائي مرتبط بالمحكمة أو المرحلة القضائية.

لا يجوز تسمية رقم ملف المكتب بـ `رقم الدعوى` حتى لا يختلط مع رقم الأساس.

---

## 2. رأس ملف الدعوى الجاري

يعرض رأس ثابت أعلى شاشة الدعوى الجارية في كل التبويبات.

البيانات:

```text
رقم ملف المكتب
نوع الدعوى
صفة / نوع المرحلة الحالية
المحكمة / المرجع
رقم الأساس وسنته للمرحلة الحالية
الموكل الرئيسي
الخصم الرئيسي
الموعد القادم
حالة الدعوى
المحامي المسؤول
المحامون المساعدون إن وجدوا
```

يبقى الرأس ظاهراً أثناء التنقل بين تبويبات الملف.

## 3. الأدوار الأساسية داخل التطبيق

يجب التمييز بين:

```text
دور الصلاحيات
الصفة الوظيفية / المهنية داخل المكتب
```

### 3.1 دور الصلاحيات

يحدد ماذا يستطيع المستخدم فعله في النظام.

أمثلة:

```text
مالك المكتب
مدير النظام
مدير مكتب
محامي
محامي مساعد
سكرتير / أمين سر
محاسب
مدخل بيانات / أرشيف
مراجع / مدقق
مشاهد فقط
```

### 3.2 الصفة المهنية داخل الملفات

تحدد علاقة المستخدم بملف معين.

أمثلة:

```text
محامي مسؤول
محامي مساعد
موظف متابع
مدخل نتيجة
مدقق مستندات
```

### 3.3 قرار الدعوى الجارية

في الدعوى الجارية نعتمد:

```text
محامٍ مسؤول واحد
محامون مساعدون اختياريون
```

المحامي المسؤول يجب أن يختار من مستخدمي التطبيق، لا كنص حر.

## 8. تبويب المراحل والجلسات

### 8.1 المراحل

كل مرحلة يجب أن تحمل صفة مركبة، مثل:

```text
صلح مدني
بداية مدنية
استئناف مدني
نقض مدني
صلح جزاء
بداية جزاء
استئناف جنح
جنايات
نقض جزائي
محكمة قضاء إداري
المحكمة الإدارية العليا
```

أي أن المرحلة لا تكون `صلح` فقط، بل يجب أن تتضمن الاختصاص أو النوع.

### 8.2 بيانات المرحلة

كل مرحلة تحتوي:

```text
نوع / صفة المرحلة
المحكمة / المرجع
المحافظة
رقم الغرفة
رقم الأساس أو رقم المرجع
السنة
تاريخ البدء
تاريخ الانتهاء
القرار / النتيجة
الوثائق المرتبطة
```

### 8.3 تحذير الحقول الثابتة

بعض الحقول يجب التعامل معها كحقول ثابتة بعد الحفظ أو تعديلها ضمن آلية خاصة فقط.

أمثلة:

```text
نوع الدعوى
نوع المرحلة
المحكمة / المرجع
رقم الأساس وسنته
رقم ملف المكتب
```

عند إدخال هذه الحقول يجب عرض تنبيه واضح مثل:

```text
تنبيه: هذا الحقل يؤثر على بنية الملف والمراحل القضائية، وقد لا يمكن تعديله لاحقاً إلا من شاشة تعديل المرحلة.
```

### 8.4 الجلسات داخل المرحلة

الجلسات تعرض داخل المرحلة التي تنتمي إليها، فلا تختلط جلسات البداية بجلسات الاستئناف أو النقض.

كل جلسة:

```text
تاريخ الجلسة
وقت الجلسة
الحضور
ما تم في الجلسة
قرار المحكمة
الإجراء المطلوب لاحقاً
الموعد القادم
ملاحظات
مرفقات
```

### 8.5 نتيجة الجلسة

حقول نتيجة الجلسة:

```text
تاريخ الجلسة
هل حضر الموكل؟
هل حضر الخصم؟
هل حضر وكيل الخصم؟
ما تم في الجلسة
قرار المحكمة
الإجراء المطلوب لاحقاً
الموعد القادم
مرفقات
ملاحظات
```

الإجراء القادم يبقى نصاً حراً مع اقتراحات جاهزة.

اقتراحات الإجراء القادم:

```text
تبليغ
مرافعة
إبراز وثائق
سماع شهود
خبرة
تفهيم حكم
رد على مذكرة
تدقيق
إمهال
مخصص
```

## 9. إغلاق / إنهاء مرحلة

تعتمد التسمية:

```text
إغلاق / إنهاء مرحلة
```

وليس فقط `إغلاق الدعوى`، لأن الدعوى قد تغلق مرحلة وتبدأ مرحلة طعن لاحقة.

### أسباب الإغلاق

```text
التخلي عن الملف / تسليمه
إسقاط / تنازل / صلح
صدور قرار قضائي
انتهاء التنفيذ
سبب آخر
```

### التخلي عن الملف

عند اختيار التخلي:

```text
تاريخ التخلي
اسم المحامي الجديد إن وجد
هل تريد إنشاء لا مانع؟
ملاحظات
```

صيغة لا مانع يجب أن تكون قابلة للتعديل قبل التصدير.

### إسقاط / تنازل / صلح

حقول:

```text
من قام بالإسقاط؟
نوع الإسقاط
تاريخ الإسقاط
مستند الإسقاط
ملاحظات
```

### صدور قرار قضائي

حقول:

```text
رقم القرار
تاريخ القرار
مضمون القرار
إرفاق القرار
```

ثم يختار المستخدم الأثر:

```text
إغلاق نهائي
فتح استئناف
فتح نقض
فتح مخاصمة
فتح تنفيذ
```

حسب المرحلة الحالية.

## 18. قرارات إضافية مثبتة بعد النقاش

### 18.1 المحامي المسؤول

- المحامي المسؤول يجب أن يختار من مستخدمي التطبيق.
- نحتاج لاحقاً إلى تسمية أدوار مهنية أساسية بجانب أدوار الصلاحيات.
- تعتمد الدعوى محامياً مسؤولاً واحداً، مع إمكانية محامين مساعدين اختيارياً.

### 18.2 حالة الدعوى

تم اعتماد الحالات التفصيلية المقترحة:

```text
قيد التحضير
مسجلة
قيد التبليغ
قيد المحاكمة
بانتظار قرار
بانتظار تنفيذ
معلقة
مؤجلة
مغلقة / منتهية
```

### 18.3 صفة المرحلة

لا يكفي أن تكون المرحلة `صلح` أو `بداية` فقط، بل يجب أن تحمل صفة الاختصاص:

```text
صلح مدني
بداية مدنية
صلح جزاء
بداية جزاء
استئناف مدني
نقض جزائي
محكمة قضاء إداري
```

### 18.4 الحقول الثابتة

الحقول التي تؤثر على بنية الملف أو المرحلة يجب أن تعرض تحذيراً قبل الحفظ:

```text
نوع الدعوى
نوع المرحلة
المحكمة / المرجع
رقم الأساس
سنة الأساس
رقم ملف المكتب
```

وتعدل لاحقاً من تبويب المراحل لا من الملخص.

### 18.5 إغلاق / إنهاء مرحلة

تم اعتماد تسمية:

```text
إغلاق / إنهاء مرحلة
```

بدلاً من `إغلاق الدعوى` فقط، لأن الإغلاق قد يكون نهاية مرحلة لا نهاية الملف كاملاً.

### 18.6 أسباب الإغلاق

```text
التخلي عن الملف / تسليمه
إسقاط / تنازل / صلح
صدور قرار قضائي
انتهاء التنفيذ
سبب آخر
```

### 18.7 التخلي ولا مانع

عند التخلي عن الملف يجب دعم تصدير كتاب `لا مانع` قابل للتعديل قبل الطباعة أو الحفظ.

## 19. القرار المعتمد — آثار الإغلاق والإجراءات اللاحقة

بعد مراجعة أثر الشطب والتجديد والرد الشكلي والتخلي عن الملف، تعتمد شاشة الدعوى القاعدة التالية في `إغلاق / إنهاء مرحلة` وفي شاشة الملفات المنتهية:

```text
شطب الدعوى = لا يعني نهاية الحق، ويجب أن يفتح مسار تجديد الدعوى.
رد الدعوى شكلاً = لا يعامل كإغلاق نهائي تلقائياً، ويجب أن يقترح استئناف/نقض/دعوى جديدة مرتبطة بحسب المرحلة والاختصاص.
التخلي عن الملف / تسليمه = إغلاق علاقة المكتب بالملف مع إمكانية إصدار لا مانع وإعادة فتح الملف إذا عاد للمكتب لاحقاً.
التنازل عن الحق = إغلاق نهائي حساس، ولا يعتمد إلا بعد تحذير واضح وصلاحية مناسبة وإرفاق مستند أو سبب.
صدور حكم أو قرار = إنهاء مرحلة، وليس بالضرورة إغلاق الملف، ويجب أن يفتح أزرار التصعيد المناسبة: استئناف، نقض، مخاصمة، تنفيذ.
```

### 19.1 الشطب وتجديد الدعوى

عند اختيار نتيجة `شطب` يجب ألا يتعامل التطبيق معها كخسارة نهائية أو نهاية للحق.

الأثر العملي في التطبيق:

```text
حالة المرحلة: مشطوبة / قابلة للتجديد
حالة الملف: منتهية مرحلياً أو معلقة حسب اختيار المكتب
الإجراء اللاحق المقترح: تجديد الدعوى
```

زر `تجديد الدعوى` ينشئ مساراً مرتبطاً بالمرحلة المشطوبة، ولا ينشئ ملفاً منفصلاً بلا علاقة.

بيانات مسار التجديد المقترحة:

```text
تاريخ الشطب
سبب الشطب إن وجد
تاريخ قرار التجديد / طلب التجديد
رقم الأساس الجديد إن وجد
سنة الأساس الجديدة إن وجدت
المحكمة / الغرفة
الرسوم / المصاريف
مرفقات طلب التجديد
ملاحظات
```

### 19.2 الرد الشكلي

`رد الدعوى شكلاً` يجب أن يبقى سبباً مستقلاً عن `رد الدعوى موضوعاً`.

الأثر العملي:

```text
لا يغلق الحق الموضوعي تلقائياً.
يعرض التطبيق خيارات لاحقة حسب المرحلة:
استئناف
نقض
دعوى جديدة مرتبطة
حفظ بلا إجراء لاحق
```

عند إنشاء دعوى جديدة مرتبطة بسبب رد شكلي، يجب حفظ صلة واضحة بالملف السابق حتى يبقى تاريخ العمل معروفاً.

### 19.3 التخلي عن الملف ولا مانع

`التخلي عن الملف / تسليمه` هو سبب إغلاق علاقة المكتب بالملف، وليس حكماً قضائياً.

يجب دعم:

```text
تاريخ التخلي
اسم المحامي الجديد إن وجد
رقم هاتف المحامي الجديد إن وجد
سبب التخلي
هل صدر لا مانع؟
رقم / تاريخ كتاب لا مانع إن اعتمد المكتب ترقيماً
نص لا مانع قابل للتعديل قبل الطباعة
مرفقات التسليم
```

يبقى الملف قابلاً لـ `إعادة فتح كملف جارٍ` إذا عاد الموكل للمكتب لاحقاً، مع حفظ سبب إعادة الفتح في سجل الأحداث.

### 19.4 التنازل والإسقاط والصلح

يجب التمييز بين:

```text
تنازل عن الدعوى / الخصومة
تنازل عن الحق
صلح / تسوية
إسقاط أو ترك
```

`التنازل عن الحق` هو الأكثر خطورة، لذلك عند اختياره يجب عرض تحذير قبل الحفظ، مثل:

```text
تنبيه: التنازل عن الحق قد يمنع المطالبة به لاحقاً. لا تحفظ هذا السبب إلا إذا كان مثبتاً بمستند أو بتوجيه واضح من الموكل.
```

ولا ينفذ إلا بصلاحية مناسبة.

### 19.5 صدور حكم أو قرار

عند صدور حكم أو قرار، يجب إنهاء المرحلة الحالية فقط، ثم عرض الخيارات المناسبة لاختصاص ومرحلة الدعوى:

```text
استئناف
طعن بالنقض
مخاصمة
فتح تنفيذ
إغلاق نهائي
حفظ بانتظار قرار الموكل
```

القائمة ليست ثابتة لكل الدعاوى، بل تتغير حسب نوع الدعوى والمرحلة والاختصاص.

### 19.6 شاشة الدعوى المنتهية

الدعوى المنتهية يجب أن تعرض تبويباً أو قسماً باسم:

```text
إجراءات لاحقة / طعون
```

ويظهر فيه فقط ما يناسب سبب الإغلاق والمرحلة، مثل:

```text
تجديد الدعوى
استئناف
طعن بالنقض
مخاصمة
فتح تنفيذ
إعادة فتح كملف جارٍ
طباعة لا مانع
```

لا تعرض النواقص التشغيلية المعتادة في الملف المنتهي، إلا إن كان الملف منتهياً مرحلياً ومطلوباً منه إجراء لاحق.

## 20. القرار المعتمد — شكل شاشة الدعوى المنتهية

تعتمد الدعوى المنتهية نفس شاشة الدعوى `Workspace` مع وضع منتهي، وليس شاشة منفصلة بالكامل.

المبدأ:

```text
نفس بنية الدعوى حتى لا يضيع سياق الملف.
معظم الحقول الحساسة تصبح قراءة فقط.
تزال النواقص التشغيلية المعتادة.
يضاف قسم / تبويب إجراءات لاحقة / طعون.
```

### 20.1 التبويبات في الدعوى المنتهية

تعتمد نفس تبويبات الدعوى الجارية تقريباً، مع تعديلين:

```text
إلغاء تبويب النواقص التشغيلية من العرض المعتاد.
إضافة تبويب إجراءات لاحقة / طعون.
```

التبويبات المقترحة:

```text
1. الملخص
2. الأطراف والوكالات
3. المراحل والجلسات
4. المستندات
5. أوامر العمل
6. المالية
7. إجراءات لاحقة / طعون
8. سجل الأحداث
```

### 20.2 وضع القراءة فقط

في الدعوى المنتهية تكون البيانات التالية قراءة فقط افتراضياً:

```text
رقم ملف المكتب
نوع الدعوى
المراحل القضائية المنتهية
المحكمة / المرجع
رقم الأساس وسنته
سبب الإغلاق
تاريخ الإغلاق
نتيجة المرحلة
```

أي تعديل على هذه البيانات يجب أن يتم عبر صلاحية خاصة أو إجراء تصحيح موثق في سجل الأحداث، وليس كتعديل عادي.

### 20.3 إعادة فتح ملف منتهٍ

إعادة فتح الدعوى المنتهية كملف جارٍ تكون محصورة مبدئياً بـ:

```text
مالك المكتب
مدير النظام / المدير المخول
```

ولا تنفذ إلا مع:

```text
سبب إعادة الفتح
تاريخ إعادة الفتح
تسجيل الحدث في سجل الأحداث
```

### 20.4 كتاب لا مانع

يعتمد كتاب `لا مانع` بطريقتين:

```text
نص افتراضي قابل للتعديل فوراً قبل الطباعة.
قابل لاحقاً للربط بقوالب النماذج القانونية لتوليده من قالب محفوظ.
```

هذا يعني أن التنفيذ الأول لا يتوقف على بناء نظام النماذج بالكامل، لكنه لا يغلق الباب أمام ربطه لاحقاً بالقوالب القانونية.

## 21. قرارات تشغيلية — المالية والتصعيد والترقيم بعد انتهاء الدعوى

### 21.1 المالية بعد انتهاء الدعوى

في الدعوى المنتهية تبقى المالية قابلة لإضافة عمليات محدودة فقط:

```text
سند قبض
مصروف مرتبط بالملف
```

أما تعديل اتفاق الأتعاب أو إعادة بناء مالية الملف فلا يكون إجراءً عادياً بعد الإغلاق، ويحتاج صلاحية إدارية أو مسار تصحيح خاص.

السبب العملي:

```text
الملف قد ينتهي قضائياً لكن تبقى أتعاب أو مصاريف أو قبض لاحق.
```

### 21.2 الاستئناف / النقض / المخاصمة من دعوى منتهية

عند الضغط على:

```text
استئناف
طعن بالنقض
مخاصمة
```

من داخل دعوى منتهية، يعتمد التطبيق مبدئياً:

```text
إضافة مرحلة جديدة داخل نفس رقم ملف المكتب.
```

أي أن رقم ملف المكتب يبقى هو الرابط الداخلي الأساسي، بينما المرحلة الجديدة تحمل رقمها القضائي الخاص:

```text
رقم أساس الاستئناف / سنة الاستئناف
رقم الطعن / سنة الطعن
رقم المخاصمة / سنتها أو مرجعها
```

هذا يحافظ على أن الملف في المكتب هو ملف واحد له مراحل متعددة، ولا يشتت العمل إلى ملفات منفصلة إلا إذا تقرر لاحقاً إعداد خاص لسياسة المكتب.

### 21.3 الدعوى الجديدة المرتبطة بعد الرد الشكلي

إذا انتهت الدعوى برد شكلي، ثم قرر المكتب إقامة دعوى جديدة مرتبطة، يعتمد رقم ملف مكتب تابع للرقم القديم بصيغة لاحقة.

مثال:

```text
2026/001-R
2026/001-تجديد
2026/001-دعوى-مرتبطة
```

الصيغة النهائية للّاحقة يجب أن تكون قابلة للضبط لاحقاً حسب سياسة المكتب، لكن القاعدة المعتمدة الآن:

```text
ليست رقماً منفصلاً بلا علاقة.
وليست نفس المرحلة القضائية القديمة.
بل ملف/مسار جديد مرتبط يحمل لاحقة واضحة.
```

## 22. القرار المعتمد — أوامر العمل داخل الدعوى المنتهية

لا تبقى أوامر العمل داخل الدعوى المنتهية مفتوحة كأنها دعوى جارية، ولا تتحول إلى قراءة فقط بالكامل.

تعتمد القاعدة الوسطية:

```text
أوامر لاحقة محدودة فقط.
```

الأوامر المسموحة في الدعوى المنتهية تكون مرتبطة بإغلاق الملف أو أثره اللاحق، مثل:

```text
طباعة لا مانع
تسليم مستندات
قبض أتعاب
إضافة مصروف مرتبط بالملف
متابعة استئناف / نقض / مخاصمة
تحضير ملف تنفيذ
أرشفة مستند لاحق
تسليم صورة حكم أو قرار
```

لا يسمح بإنشاء أوامر عمل تشغيلية عادية بلا سبب، حتى لا تعود الدعوى المنتهية عملياً إلى ملف جارٍ من دون إجراء إعادة فتح واضح.

إذا كان المطلوب متابعة عادية أو جلسات أو مواعيد تشغيلية كاملة، يجب اختيار أحد المسارات التالية:

```text
إعادة فتح كملف جارٍ بصلاحية المالك/المدير.
فتح مرحلة طعن داخل نفس الملف.
فتح تنفيذ.
فتح دعوى جديدة مرتبطة عند الرد الشكلي أو الحاجة القانونية.
```

كل أمر عمل لاحق داخل ملف منتهٍ يجب أن يظهر في سجل الأحداث، وأن يحمل سبباً أو نوعاً واضحاً يبيّن أنه أمر لاحق بعد الإغلاق.

## 23. القرار المعتمد — ويزارد إغلاق / إنهاء مرحلة

يعتمد زر المرحلة اسم:

```text
إغلاق / إنهاء مرحلة
```

ولا يعتمد اسم `إغلاق الدعوى` فقط، لأن انتهاء المرحلة قد يؤدي إلى طعن أو تنفيذ أو تجديد أو ملف مرتبط، وليس بالضرورة إلى نهاية الملف نهائياً.

### 23.1 مكان الزر

يوضع الزر داخل تبويب:

```text
المراحل والجلسات
```

وضمن بطاقة المرحلة الحالية تحديداً، بجانب أزرار مثل:

```text
إضافة جلسة
إدخال نتيجة جلسة
إغلاق / إنهاء مرحلة
```

ولا يوضع زر الإغلاق في كل مكان داخل الدعوى منعاً للخطأ.

### 23.2 شاشة البداية

قبل البدء يعرض الويزارد ملخص المرحلة التي سيتم إنهاؤها:

```text
رقم ملف المكتب
نوع الدعوى
المرحلة الحالية
المحكمة
رقم الأساس
سنة الأساس
آخر جلسة
آخر إجراء
الموعد القادم إن وجد
المحامي المسؤول
```

مع تنبيه واضح:

```text
أنت على وشك إنهاء المرحلة الحالية.
قد يؤدي ذلك إلى نقل الملف إلى المنتهي، أو فتح مرحلة طعن، أو إنشاء إجراء لاحق حسب سبب الإنهاء.
```

### 23.3 أسباب إنهاء المرحلة

تعتمد القائمة التالية:

```text
صدور حكم / قرار قضائي
شطب الدعوى
رد الدعوى شكلاً
إسقاط / تنازل / صلح
انتهاء التنفيذ
التخلي عن الملف / تسليمه
حفظ / تعليق بانتظار قرار الموكل
سبب آخر
```

كل سبب يفتح حقولاً خاصة به، ولا تستخدم نفس الحقول لجميع الأسباب.

### 23.4 صدور حكم / قرار قضائي

الحقول:

```text
نوع القرار
رقم القرار
تاريخ القرار
تاريخ التبليغ إن وجد
مضمون القرار
النتيجة بالنسبة للموكل
إرفاق صورة الحكم / القرار
ملاحظات
```

أنواع القرار المقترحة:

```text
حكم
قرار
قرار إعدادي
قرار قطعي
قرار مبرم
قرار قابل للطعن
قرار آخر
```

النتيجة بالنسبة للموكل:

```text
رابح
خاسر
رابح جزئياً
خاسر جزئياً
غير محدد
```

لا تثبت مواعيد الطعن كقواعد جامدة في المرحلة الأولى، بل تعتمد حقول عملية:

```text
هل يوجد ميعاد طعن؟
تاريخ بدء الميعاد
آخر يوم للطعن
ملاحظات على الميعاد
```

بعد الحفظ تظهر الخيارات المناسبة حسب المرحلة والاختصاص:

```text
إغلاق نهائي
فتح استئناف
فتح طعن بالنقض
فتح مخاصمة
فتح تنفيذ
حفظ بانتظار قرار الموكل
```

### 23.5 شطب الدعوى

الحقول:

```text
تاريخ الشطب
سبب الشطب
من كان حاضراً؟
هل حضر الموكل؟
هل حضر الخصم؟
هل حضر وكيل الخصم؟
رقم ضبط الجلسة إن وجد
مرفق قرار / ضبط الشطب
ملاحظات
```

أسباب الشطب:

```text
غياب الطرفين
غياب المدعي / طالب الدعوى
سبب إجرائي
سبب آخر
```

الأثر:

```text
حالة المرحلة: مشطوبة / قابلة للتجديد
الإجراء اللاحق المقترح: تجديد الدعوى
```

أزرار الإجراءات اللاحقة:

```text
تجديد الدعوى
إغلاق بلا متابعة
إعادة فتح كملف جارٍ
```

حقول تجديد الدعوى:

```text
تاريخ طلب التجديد
المحكمة
الغرفة
رقم الأساس الجديد إن وجد
سنة الأساس الجديدة إن وجدت
الرسوم
مرفق طلب التجديد
ملاحظات
```

### 23.6 رد الدعوى شكلاً

الحقول:

```text
تاريخ القرار
رقم القرار
سبب الرد الشكلي
مضمون القرار
هل يمكن تدارك السبب؟
إرفاق القرار
ملاحظات
```

أسباب الرد الشكلي المقترحة:

```text
عدم صحة الخصومة
عدم الاختصاص
نقص وكالة
نقص صفة
نقص مصلحة
خطأ في الشكل
عدم استيفاء رسم / إجراء
سبب آخر
```

هل يمكن تدارك السبب؟

```text
نعم
لا
غير مؤكد
```

بعد الحفظ تظهر:

```text
استئناف
طعن بالنقض
دعوى جديدة مرتبطة
حفظ بلا إجراء لاحق
```

الدعوى الجديدة المرتبطة تقترح رقم ملف مكتب تابعاً مثل:

```text
2026/001-R
2026/001-تجديد
```

مع حفظ الربط بين الملفين.

### 23.7 إسقاط / تنازل / صلح

الحقول:

```text
نوع الإنهاء
من قام بالإسقاط / التنازل / الصلح؟
تاريخ الإنهاء
هل يوجد مستند؟
إرفاق المستند
مضمون الإنهاء
ملاحظات
```

أنواع الإنهاء:

```text
تنازل عن الدعوى / الخصومة
تنازل عن الحق
صلح / تسوية
إسقاط
ترك
```

عند اختيار `تنازل عن الحق` يظهر تحذير واضح:

```text
تنبيه هام:
التنازل عن الحق قد يمنع المطالبة به لاحقاً.
لا تعتمد هذا السبب إلا إذا كان مثبتاً بمستند أو بتوجيه واضح من الموكل.
```

بعد الحفظ:

```text
إغلاق نهائي
حفظ مع إمكانية مراجعة
طباعة ملخص الإنهاء
```

وفي حالة الصلح / التسوية يمكن عرض:

```text
إضافة سند قبض
إضافة مصروف
تسجيل التزام مالي لاحق
```

### 23.8 انتهاء التنفيذ

الحقول:

```text
تاريخ انتهاء التنفيذ
طريقة الانتهاء
المبلغ المحصل إن وجد
المصاريف
هل تم تسليم المبلغ للموكل؟
إرفاق مستند التنفيذ / المخالصة
ملاحظات
```

طرق الانتهاء:

```text
تحصيل كامل
تحصيل جزئي
تعذر التنفيذ
مصالحة تنفيذية
إغلاق ملف التنفيذ
سبب آخر
```

بعد الحفظ:

```text
إغلاق نهائي
إضافة سند قبض
إضافة مصروف
طباعة كشف مالي
طباعة ملخص التنفيذ
```

### 23.9 التخلي عن الملف / تسليمه

الحقول:

```text
تاريخ التخلي
سبب التخلي
اسم المحامي الجديد إن وجد
رقم هاتف المحامي الجديد
هل تم تسليم المستندات؟
هل تريد إنشاء لا مانع؟
مرفقات التسليم
ملاحظات
```

أسباب التخلي:

```text
طلب الموكل
انتقال الملف لمحامٍ آخر
عدم الاتفاق على الأتعاب
تعذر المتابعة
سبب إداري
سبب آخر
```

إذا اختار المستخدم إنشاء `لا مانع`، يفتح نص افتراضي قابل للتعديل قبل الطباعة، مع قابلية ربطه لاحقاً بالنماذج القانونية.

بعد الحفظ:

```text
طباعة لا مانع
تسليم مستندات
إضافة سند قبض
إضافة مصروف
إعادة فتح كملف جارٍ - للمالك/المدير فقط
```

### 23.10 حفظ / تعليق بانتظار قرار الموكل

هذا ليس إغلاقاً نهائياً.

الحقول:

```text
سبب التعليق
تاريخ التعليق
آخر موعد للمراجعة
الشخص المطلوب مراجعته
ملاحظات
```

أسباب التعليق:

```text
بانتظار قرار الموكل بالطعن
بانتظار دفع الرسوم
بانتظار إحضار مستند
بانتظار مصالحة
بانتظار تعليمات
سبب آخر
```

الأثر:

```text
حالة الملف: معلق / بانتظار قرار الموكل
يبقى ضمن الملفات الجارية إذا كان له موعد مراجعة
```

### 23.11 سبب آخر

الحقول:

```text
عنوان السبب
شرح السبب
تاريخ الإنهاء
هل يعتبر إغلاقاً نهائياً؟
هل يحتاج إجراء لاحق؟
مرفقات
ملاحظات
```

إذا احتاج إجراء لاحقاً، يطلب:

```text
الإجراء اللاحق المطلوب
تاريخ المتابعة
```

### 23.12 شاشة المراجعة قبل الحفظ

قبل التنفيذ النهائي يعرض الويزارد:

```text
المرحلة التي سيتم إنهاؤها
سبب الإنهاء
تاريخ الإنهاء
الأثر على الملف
الإجراء اللاحق
المرفقات
```

مع أزرار:

```text
حفظ وتنفيذ
رجوع للتعديل
إلغاء
```

### 23.13 أثر الحفظ

حسب الاختيار:

```text
إغلاق نهائي:
حالة المرحلة: منتهية
حالة الملف: منتهٍ
ينتقل للملفات المنتهية
```

```text
فتح استئناف / نقض / مخاصمة:
حالة المرحلة القديمة: منتهية
تضاف مرحلة جديدة داخل نفس رقم ملف المكتب
حالة الملف: جارٍ
```

```text
تجديد دعوى مشطوبة:
تبقى المرحلة القديمة مشطوبة
تضاف حركة تجديد مرتبطة بها
يبقى رقم ملف المكتب نفسه أو تابعاً حسب الحالة
```

```text
دعوى جديدة مرتبطة بعد رد شكلي:
ينشأ رقم ملف مكتب تابع
يرتبط بالملف القديم
تنقل الأطراف والوكالات والمستندات حسب اختيار المستخدم لاحقاً
```

```text
التخلي عن الملف:
حالة الملف: منتهٍ بسبب التخلي
يظهر زر طباعة لا مانع
تبقى إمكانية قبض أتعاب أو تسجيل مصروف
إعادة الفتح فقط للمالك/المدير
```

### 23.14 سجل الأحداث والمستندات

كل عملية في الويزارد يجب أن تسجل في Timeline.

أمثلة:

```text
تم إنهاء مرحلة بداية مدنية بسبب صدور حكم.
تم فتح مرحلة استئناف مدني.
تم شطب الدعوى.
تم إنشاء طلب تجديد الدعوى.
تم إغلاق الملف بسبب التخلي وتسليم المستندات.
تم إصدار كتاب لا مانع.
```

المستندات الناتجة أو المرتبطة تذهب إلى تبويب المستندات بتصنيف مناسب، مثل:

```text
صورة الحكم
ضبط الجلسة
قرار الشطب
طلب التجديد
كتاب لا مانع
مستند صلح
مخالصة
كشف مالي
```

### 23.15 خلاصة الويزارد

يعتمد الويزارد المنطق التالي:

```text
ما المرحلة التي سننهيها؟
لماذا انتهت؟
ما أثر ذلك قانونياً وعملياً؟
هل الملف ينتهي أم ينتقل لمرحلة جديدة؟
هل يوجد إجراء لاحق؟
ما المستندات التي تثبت ذلك؟
سجل كل شيء في سجل الأحداث.
```
