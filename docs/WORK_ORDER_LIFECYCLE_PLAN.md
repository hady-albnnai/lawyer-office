# خطة أوامر العمل ودورة حياتها

تاريخ الاعتماد: 2026-07-20

هذه الخطة تضبط معنى أمر العمل داخل التطبيق، ودورة حياته، وعلاقته بعمل اليوم والغد والملفات والنتائج.

---

## 1. تعريف أمر العمل

أمر العمل هو مهمة تنفيذية داخل مكتب العمل.

قد يرتبط بـ:

```text
دعوى
إجراء / معاملة
عقد
شركة
وكالة
```

وقد يكون:

```text
مهمة عامة غير مرتبطة بملف
```

أمر العمل ليس ملفاً مستقلاً، بل مهمة داخل ملف أو ضمن مكتب العمل.

---

## 2. الفرق بين أمر العمل والإجراء / المعاملة

### 2.1 الإجراء / المعاملة

الإجراء ملف مستقل مثل:

```text
إجراء/2026/0004 — حصر إرث
إجراء/2026/0005 — معاملة فراغ عقاري
```

وله:

```text
جهة
رقم معاملة
مستندات
نتيجة
مراجعات
روابط بملفات أخرى
```

### 2.2 أمر العمل

أمر العمل مهمة داخل ملف أو مكتب العمل، مثل:

```text
مراجعة السجل العقاري يوم الثلاثاء
دفع رسم
الاتصال بالموكل
تحضير مذكرة
إحضار صورة وكالة
```

القاعدة:

```text
الإجراء = ملف.
أمر العمل = مهمة.
```

---

## 3. حالات أمر العمل

تعتمد دورة الحياة التالية:

```text
مفتوح
نتيجة مدخلة
معتمد
مؤجل
ملغى
يحتاج تصحيح
```

### 3.1 مفتوح

```text
المهمة لم تنفذ بعد أو لم تدخل نتيجتها.
```

### 3.2 نتيجة مدخلة

```text
تم إدخال نتيجة المهمة، لكنها لم تعتمد بعد إذا كان الاعتماد مطلوباً.
```

### 3.3 معتمد

```text
تمت مراجعة النتيجة واعتماد أثرها على الملف.
```

### 3.4 مؤجل

```text
تم تأجيل المهمة إلى تاريخ لاحق مع سبب.
```

### 3.5 ملغى

```text
تم إلغاء المهمة مع سبب.
```

### 3.6 يحتاج تصحيح

```text
تم رفض النتيجة أو طلب تعديلها قبل اعتمادها.
```

## 6. المسؤول والمنفذ

يجب الفصل بين:

```text
المسؤول / المشرف
المنفذ / المكلف
```

### 6.1 المسؤول / المشرف

يكون غالباً مستخدماً داخل التطبيق:

```text
مالك المكتب
مدير المكتب
محامي أستاذ
```

### 6.2 المنفذ / المكلف

يمكن أن يكون:

```text
محامي متمرن من القائمة المرجعية
معقب / مندوب من القائمة المرجعية
مستخدم داخلي
```

---

## 7. نتيجة أمر العمل

عند إدخال نتيجة أمر العمل، تحفظ البيانات التالية:

```text
تاريخ النتيجة الفعلي
المنفذ الفعلي
مدخل النتيجة
مضمون النتيجة
مرفقات
مصروف أو قبض إن وجد
موعد قادم إن وجد
إجراء قادم إن وجد
هل تحتاج اعتماد؟
ملاحظات
```

---

## 8. اعتماد النتائج

تعتمد القاعدة التالية:

```text
اعتماد نتيجة أمر العمل يكون حسب صلاحية المستخدم.
```

القواعد المقترحة:

```text
مالك المكتب يمكنه الاعتماد.
محامي أستاذ يمكنه الاعتماد.
مدير المكتب يمكنه الاعتماد إذا منحه المالك صلاحية اعتماد النتائج.
```

إذا أدخل النتيجة مستخدم مخول، يمكن أن يختار:

```text
حفظ واعتماد
```

أما إذا أدخلها مستخدم غير مخول أو مستخدم يحتاج مراجعة، تبقى الحالة:

```text
نتيجة مدخلة
```

حتى يعتمدها مستخدم مخول.

---

## 9. أثر اعتماد النتيجة

عند اعتماد النتيجة، يمكن أن تؤدي إلى:

```text
إضافة نتيجة جلسة
إضافة خطوة في إجراء / معاملة
إضافة مستند
إضافة مصروف
إضافة سند قبض
إنشاء موعد جديد
إنشاء أمر عمل جديد
إغلاق نقص
فتح نقص جديد
تحديث حالة الملف
إغلاق / إنهاء مرحلة
فتح ملف مرتبط
تسجيل حدث في سجل الأحداث
```

لا يجب أن تحدث الآثار الحساسة قبل الاعتماد إذا كانت النتيجة تحتاج اعتماداً.

---

## 10. أمر العمل وعمل اليوم والغد

### 10.1 ظهور أمر العمل في عمل الغد

إذا كان:

```text
تاريخ العمل = الغد
```

يظهر أمر العمل في:

```text
عمل الغد
```

ويخضع للتوزيع والإسناد والطباعة والإرسال عبر واتس اب.

### 10.2 ظهور أمر العمل في أعمال اليوم

إذا كان:

```text
تاريخ العمل = اليوم
```

يظهر في:

```text
أعمال اليوم
```

ويصبح قابلاً لإدخال النتيجة.

### 10.3 المهام غير المكتملة

إذا مر تاريخ العمل ولم تدخل النتيجة ولم يتم التأجيل أو الإلغاء، ينتقل العمل إلى:

```text
مهام غير مكتملة
```

ولا يرحل تلقائياً إلى اليوم الجديد أو الغد.

---

## 11. أمر العمل العام

يسمح النظام بأمر عمل عام لا يرتبط بملف.

أمثلة:

```text
شراء قرطاسية
مراجعة البنك
اجتماع داخلي
اتصال إداري
عمل تنظيمي داخل المكتب
```

أمر العمل العام يظهر في مكتب العمل، لكنه لا يظهر ضمن ملفات المكتب لأنه غير مرتبط بملف قانوني.

---

## 12. عدد المنفذين

في المرحلة الحالية يعتمد أمر العمل:

```text
منفذ رئيسي واحد
```

ويمكن لاحقاً إضافة مساعدين عند الحاجة.

الغرض من ذلك منع تعقيد التوزيع والطباعة والإرسال في النسخة الأولى.

---

## 13. النتيجة المالية لأمر العمل

يمكن أن تتضمن نتيجة أمر العمل أثراً مالياً، مثل:

```text
دفع رسم
سند قبض
مصروف انتقال
مصروف تصوير
إيصال رسمي
```

القاعدة:

```text
إضافة الأثر المالي تتم إذا كان مدخل النتيجة يملك صلاحية مالية، أو بعد اعتماد مستخدم مخول مالياً.
```

لا يجوز إنشاء قيد مالي نهائي من نتيجة غير معتمدة إذا كانت تحتاج اعتماداً.

---

## 14. إلغاء وتأجيل أمر العمل

### 14.1 التأجيل

عند التأجيل يجب إدخال:

```text
تاريخ جديد
سبب التأجيل
ملاحظات
```

ويظهر العمل في التاريخ الجديد.

### 14.2 الإلغاء

عند الإلغاء يجب إدخال:

```text
سبب الإلغاء
تاريخ الإلغاء
من ألغى
ملاحظات
```

ولا يحذف أمر العمل، بل تحفظ حالته:

```text
ملغى
```

```text
إلغاء

```

## 16. الخلاصة التنفيذية

القواعد المعتمدة:

```text
أمر العمل مهمة وليس ملفاً.
الإجراء / المعاملة ملف مستقل، وأمر العمل مهمة داخله أو خارجه.
حالات أمر العمل: مفتوح، نتيجة مدخلة، معتمد، مؤجل، ملغى، يحتاج تصحيح.
أنواع أوامر العمل افتراضية وقابلة للتوسع.
الاعتماد حسب صلاحية المستخدم.
أمر العمل يظهر في عمل الغد أو اليوم حسب تاريخ العمل.
أمر العمل غير المنجز ينتقل إلى مهام غير مكتملة بعد تغير التاريخ.
أمر العمل العام مسموح.
المنفذ الرئيسي واحد في المرحلة الحالية.
الأثر المالي ممكن بعد الصلاحية أو الاعتماد.
كل تغيير يسجل في سجل الأحداث.
```
