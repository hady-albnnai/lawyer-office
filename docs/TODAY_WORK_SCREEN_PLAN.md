# خطة شاشة أعمال اليوم والمهام غير المكتملة

تاريخ الاعتماد: 2026-07-20

هذه الخطة تضبط شاشة `أعمال اليوم` داخل مكتب العمل، بناءً على القاعدة التشغيلية المعتمدة:

```text
عمل الغد = تحضير وتوزيع.
عمل اليوم = تحصيل نتائج ومعالجة ما تم.
```

ملاحظة تصميمية مهمة:

```text
التصميم الشكلي النهائي للواجهات يؤجل إلى ما بعد استكمال البرمجة والهيكل الوظيفي الأساسي.
هذه الوثيقة تضبط المنطق والبيانات والسلوك، لا الشكل البصري النهائي.
```

---

## 1. تعريف أعمال اليوم

أعمال اليوم هي الأعمال التي يجب تحصيل نتائجها في تاريخ اليوم.

تتكون من:

```text
كل ما كان في عمل الغد وأصبح تاريخه اليوم.
كل ما أضافه المستخدم اليوم بتاريخ اليوم.
```

وظيفتها الأساسية ليست التحضير، بل:

```text
إدخال النتائج
معالجة النتائج
فرز آثار النتائج إلى الملفات والمستندات والمالية والمواعيد وسجل الأحداث
```

---

## 2. طريقة عرض شاشة اليوم

تعرض شاشة اليوم الأعمال حسب تقسيم التطبيق وأنواع الملفات:

```text
دعاوى
إجراءات / معاملات
عقود
شركات
وكالات
مهام عامة عند الحاجة
```

ولا يظهر في شاشة اليوم إلا القسم الذي يحتوي أعمالاً فعلية.

مثال:

إذا كان اليوم يحتوي:

```text
دعاوى
إجراءات / معاملات
عقود
```

تظهر هذه الأقسام فقط، ولا تظهر أقسام الشركات أو الوكالات إذا لم يكن فيها عمل اليوم.

---

## 3. شاشة اليوم الرئيسية

تعرض:

```text
التاريخ
تنبيه المهام غير المكتملة إن وجد
أقسام الأعمال الموجودة فقط
عدد الأعمال داخل كل قسم
```

مثال:

```text
أعمال اليوم — 2026/07/21

مهام غير مكتملة من أيام سابقة: 7

دعاوى: 5
إجراءات / معاملات: 3
عقود: 1
```

## 4. قسم الدعاوى في أعمال اليوم

عند الضغط على `دعاوى`، يفتح عرض خاص بالدعاوى يشبه أجندة المحامي.

البيانات المقترحة:

```text
المحكمة
الغرفة
رقم الأساس
سنة الأساس
الموكل
الخصم
موضوع الدعوى
الإجراء المطلوب
الموعد / الوقت
المسؤول / المشرف
المنفذ / المكلف
ملاحظات
إدخال النتيجة
```

مثال:

```text
البداية المدنية الثالثة في دمشق
أساس 120 / 2026
الموكل: أحمد
الخصم: خالد
الموضوع: مطالبة مالية
الإجراء المطلوب: إبراز مذكرة
المنفذ: المحامي المتمرن سامر
```

## 9. إدخال النتيجة

في نهاية كل عمل يظهر خيار:

```text
إدخال النتيجة
```

وتظهر نتائج مناسبة حسب نوع العمل.

### 9.1 نتائج الدعاوى

```text
تأجيل
شطب
حجز للحكم
صدور حكم / قرار
إبراز مذكرة
إبراز مستند
سماع شاهد
خبرة
غياب
ترك للمراجعة
لم تتم الجلسة
نتيجة مخصصة
```

### 9.2 نتائج الإجراءات / المعاملات

```text
تمت المراجعة
تم تقديم الطلب
تم دفع الرسم
ورد نقص
تمت الموافقة
تم الرفض
تم استلام الوثيقة
تم التأجيل
لم تتم المراجعة
نتيجة مخصصة
```

### 9.3 نتائج العقود

```text
تم التوقيع
لم يتم التوقيع
وردت ملاحظات
تم إرسال نسخة
تم استلام نسخة معدلة
تحول إلى نزاع
تم التأجيل
نتيجة مخصصة
```

### 9.4 نتائج الشركات

```text
تم تقديم الطلب
تم إرفاق وثيقة
ورد نقص
صدر قرار
تم التسجيل
تم دفع رسم
تم إنجاز تعديل
تم التأجيل
نتيجة مخصصة
```

### 9.5 نتائج الوكالات

```text
تم تنظيم الوكالة
تم تصديقها
تم استخراج صورة
تم توطين وكالة خارجية
تم العزل
تم الاعتزال
ورد نقص
تم التأجيل
نتيجة مخصصة
```

## 10. حقول نتيجة العمل

بعد اختيار النتيجة، تعرض حقول مختصرة حسب الحاجة:

```text
تفاصيل النتيجة
تاريخ النتيجة الفعلي
مرفقات
مصروف إن وجد
موعد قادم إن وجد
إجراء قادم إن وجد
ملاحظات
```

كما يجب حفظ:

```text
المنفذ الفعلي
مدخل النتيجة
وقت إدخال النتيجة
```

مثال:

```text
المنفذ الفعلي: المعقب أبو أحمد
مدخل النتيجة: مدير المكتب
```

## 11. معالجة النتيجة وفرزها

بعد حفظ النتيجة، يقوم التطبيق بفرز آثارها إلى مكانها الصحيح:

```text
الملف المرتبط
سجل الأحداث
المستندات عند وجود مرفقات
المالية عند وجود مصروف أو قبض
المواعيد عند وجود موعد قادم
النواقص عند ظهور نقص
أوامر العمل عند الحاجة
إغلاق / إنهاء مرحلة عند النتائج التي تستدعي ذلك
فتح ملف مرتبط عند الحاجة
```

أمثلة:

```text
نتيجة جلسة دعوى تحفظ داخل تبويب المراحل والجلسات في الدعوى.
نتيجة مراجعة إجراء تحفظ كخطوة / مراجعة داخل ملف الإجراء.
نتيجة توقيع عقد تحدث حالة العقد وتربط النسخة الموقعة.
نتيجة تسجيل شركة تحدث ملف الشركة وتضيف وثيقة السجل.
نتيجة تصديق وكالة تحدث حالة الوكالة وتضيف المستند.
```

## 12. المهام غير المكتملة

### 12.1 تعريفها

المهام غير المكتملة هي:

```text
أعمال كان تاريخها اليوم أو قبل اليوم
ولم تدخل نتيجتها
ولم يتم تأجيلها
ولم يتم إلغاؤها
```

### 12.2 انتقالها التلقائي

عند تغير التاريخ تلقائياً، تنتقل كل أعمال اليوم التي لم تدخل نتيجتها إلى:

```text
مهام غير مكتملة
```

ولا ترحل تلقائياً إلى اليوم الجديد أو إلى الغد.

### 12.3 سبب القاعدة

الغرض هو منع ضياع المسؤولية.

```text
المهمة التي لم تدخل نتيجتها لا تختفي.
ولا تصبح تلقائياً عمل اليوم الجديد.
بل تبقى في قسم واضح باسم مهام غير مكتملة حتى يعالجها المستخدم.
```

## 13. عرض المهام غير المكتملة

في أعلى شاشة اليوم يظهر تنبيه عند وجود مهام غير مكتملة:

```text
يوجد 7 مهام غير مكتملة من أيام سابقة
```

الأزرار:

```text
عرض المهام غير المكتملة
معالجة
طباعة عند الحاجة
```

## 14. معالجة مهمة غير مكتملة

لكل مهمة غير مكتملة خيارات:

```text
إدخال نتيجة متأخرة
ترحيل إلى اليوم
ترحيل إلى الغد
تأجيل لتاريخ محدد
إلغاء بسبب
فتح الملف
```

### 14.1 إدخال نتيجة متأخرة

يطلب:

```text
تاريخ النتيجة الفعلي
سبب التأخير في الإدخال
النتيجة
المرفقات / الملاحظات
```

مثال:

```text
الجلسة تمت أمس لكن لم تدخل النتيجة في وقتها.
```

### 14.2 الترحيل أو التأجيل

إذا لم تنفذ المهمة فعلاً، يمكن:

```text
ترحيل إلى اليوم
ترحيل إلى الغد
تأجيل لتاريخ محدد
```

مع سبب:

```text
لم يتم الذهاب
الجهة مغلقة
نقص مستندات
طلب الموكل التأجيل
سبب آخر
```

## 15. ما لا يجب أن يحدث

لا يسمح بـ:

```text
إخفاء عمل اليوم إذا لم تدخل نتيجته.
ترحيل العمل تلقائياً إلى الغد دون سبب.
خلط المهام غير المكتملة مع أعمال اليوم الجديدة دون تمييز.
إدخال نتيجة عامة لا تحفظ في الملف المرتبط.
إدخال نتيجة لا تسجل في سجل الأحداث.
```

---

## 16. ملاحظة حول التصميم الشكلي

تم الاتفاق على أن التصميم الشكلي النهائي للواجهات يؤجل إلى ما بعد استكمال البرمجة وتجهيز الوظائف الأساسية.

السبب:

```text
الواجهات الحالية غير مرضية للمستخدم.
الأولوية الآن لضبط المنطق والهيكل والبيانات.
بعد اكتمال البرمجة الأساسية، يتم الاتفاق على التصميم النهائي للواجهات بشكل مستقل.
```

---

## 17. الخلاصة التنفيذية

القواعد المعتمدة:

```text
أعمال اليوم تعرض حسب تقسيم التطبيق: دعاوى، إجراءات، عقود، شركات، وكالات.
لا يظهر إلا القسم الذي يحتوي أعمالاً فعلية.
كل قسم يعرض بيانات مناسبة لنوعه، مثل أجندة المحامي للدعاوى.
إدخال النتيجة يظهر في نهاية كل مهمة وبحسب نوعها.
النتيجة تفرز تلقائياً إلى الملف والمستندات والمالية والمواعيد وسجل الأحداث.
عند تغير التاريخ، الأعمال بلا نتيجة تنتقل إلى مهام غير مكتملة.
المهام غير المكتملة تعالج يدوياً ولا ترحل تلقائياً.
التصميم الشكلي النهائي يؤجل لما بعد اكتمال البرمجة الأساسية.
```
