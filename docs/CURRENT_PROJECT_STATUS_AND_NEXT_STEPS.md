## 4. أهم القرارات المعتمدة

### 4.1 ملفات المكتب

```text
الملفات الجارية = لوحة تشغيل يومية.
الملفات المنتهية = أرشيف بحث وإجراءات لاحقة.
إدخال الأرشيف القديم = مسار مستقل.
إنشاء الجديد = مكتب العمل فقط.
```

### 4.2 الترقيم

تم اعتماد ترقيم تلقائي عربي حسب نوع الملف:

```text
دعوى/2026/0001
إجراء/2026/0001
عقد/2026/0001
شركة/2026/0001
وكالة/2026/0001
```

والرقم غير قابل للتعديل بعد الحفظ.

### 4.3 مكتب العمل

```text
عمل الغد = تحضير وتوزيع.
عمل اليوم = تحصيل نتائج ومعالجة.
المهام غير المكتملة = أعمال لم تدخل نتيجتها بعد تغير التاريخ.
أوامر العمل = مهام وليست ملفات.
التقويم = مفكرة أعمال حسب التاريخ.
إنشاء جديد = بوابة العمل الجديد فقط.
```

### 4.4 الأدوار والتوزيع

الأدوار الافتراضية:

```text
مالك المكتب
مدير المكتب
محامي أستاذ
```

أما:

```text
المحامون المتمرنون
المعقبون / مندوبي المعاملات
```

فيدارون كقوائم مرجعية لتوزيع العمل، وليسوا مستخدمين افتراضيين.

### 4.5 المالية

```text
اتفاق أتعاب ≠ سند قبض.
مبالغ لحساب الموكل ≠ أتعاب المكتب.
لا حذف مالي نهائي.
الصندوق يبدأ بصندوق رئيسي مع إمكانية إضافة صناديق / حسابات.
```

### 4.6 المستندات

المستندات ليست بنداً رئيسياً مستقلاً، بل طبقة داخل الملفات:

```text
الدعوى: المستندات.
الإجراء: المستندات.
العقد: نسخة العقد / المستندات.
الشركة: الثبوتيات والمستندات.
الوكالة: بيانات السند والتصديقات.
المالية: مرفقات الحركة المالية.
```

### 4.7 الترخيص التجاري والشبكة

```text
المرحلة الأولى: تفعيل Offline بملف ترخيص موقّع.
المرحلة الثانية لاحقاً: تحقق Online دوري.
النسخة الفردية: SQLite محلي.
النسخة الشبكية: Local Server + PostgreSQL + مرفقات مركزية.
لا تستخدم SQLite عبر مجلد شبكة مشترك.
```

## 8. المتبقي قبل خطة التنفيذ البرمجية النهائية

```text
1. متابعة جمع المحتوى: اجتهادات النقض، مجلة المحامون، التعاميم والقرارات، الأبحاث.
2. تنظيف وفهرسة المحتوى الخام الحالي.
3. مراجعة قانونية وحقوقية للنماذج المرشحة للنسخة الأولى.
4. فحص الفجوات التقنية في الكود الحالي بعد التغييرات الأخيرة.
5. إعداد خطة التنفيذ البرمجي المرحلي النهائية.
```
