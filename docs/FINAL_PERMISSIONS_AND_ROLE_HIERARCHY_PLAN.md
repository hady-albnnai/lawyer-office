# خطة الصلاحيات النهائية وهرمية الأدوار

تاريخ الاعتماد: 2026-07-20

هذه الخطة تضبط هرم الأدوار والصلاحيات النهائية داخل تطبيق ميزان، بما يناسب المحامي الفرد والمكاتب الصغيرة وشركات المحاماة الكبيرة، مع حماية الصلاحيات المالية والإدارية الحساسة.

---

## 1. القاعدة العامة

إدارة الصلاحيات داخل التطبيق تقوم على وظيفتين:

```text
إدارة الأدوار
تحديد صلاحيات كل دور
```

الدور هو حاوية صلاحيات، ولا يكتسب أي معنى عملي إلا من خلال الصلاحيات المرتبطة به.

---

## 2. الأدوار الافتراضية المعتمدة

الأدوار الافتراضية داخل التطبيق هي فقط:

```text
مالك المكتب
مدير المكتب
محامي أستاذ
```

ولا يتم إنشاء أدوار افتراضية مثل المحامي المتمرن أو المعقب، لأن هؤلاء يدارون كقوائم مرجعية لتوزيع العمل، لا كمستخدمين افتراضيين في المرحلة الحالية.

---

## 3. مالك المكتب

مالك المكتب هو صاحب كل الصلاحيات.

يملك حصراً:

```text
إدارة المستخدمين بالكامل
إدارة الأدوار والصلاحيات
تعيين مالك مكتب
تعيين مدير مكتب
تعديل صلاحيات مدير المكتب
إدارة الصلاحيات الحساسة
النسخ الاحتياطي والاستعادة
إعادة فتح ملف منتهٍ
إلغاء أو تصحيح القيود المالية الحساسة
إدارة مبالغ لحساب الموكل
إدارة الصندوق
إدارة الإعدادات المالية
تصدير كامل البيانات
مسح بيانات التشغيل
إدارة القوائم المرجعية الحساسة
```

### قواعد حماية المالك

```text
لا يجوز حذف آخر مالك مكتب.
لا يجوز تعطيل آخر مالك مكتب.
لا يجوز لمدير المكتب تعديل صلاحيات مالك المكتب.
لا يجوز لمدير المكتب إنشاء مالك مكتب جديد.
لا يجوز لأي مستخدم غير مالك المكتب منح صلاحيات السيادة.
```

---

## 4. مدير المكتب

مدير المكتب دوره تشغيلي وإداري، وليس مالكاً للنظام.

يمكنه افتراضياً إدارة:

```text
ملفات المكتب
مكتب العمل
توزيع عمل الغد
أوامر العمل
القوائم المرجعية غير الحساسة
المحامين المتمرنين
المعقبين / مندوبي المعاملات
التقارير العامة
المستندات
الأرشيف
```

يمكنه تعيين وإدارة مستخدمين أدنى منه فقط، مثل:

```text
محامي أستاذ
```

ولا يستطيع تعيين أو إدارة:

```text
مالك مكتب
مدير مكتب آخر
دور يحتوي صلاحيات أعلى من صلاحياته
```

---

## 5. محامي أستاذ

محامي أستاذ هو المستخدم القانوني الأساسي داخل التطبيق.

يمكنه حسب الصلاحيات الممنوحة:

```text
عرض الملفات
إنشاء ملفات
إدخال نتائج
إضافة مستندات
إضافة أوامر عمل
طباعة تقارير ملف
إدارة الملفات المكلف بها
إضافة اتفاق أتعاب إذا سمح المالك
```

ولا يملك افتراضياً:

```text
إدارة المستخدمين
إدارة الأدوار والصلاحيات
إدارة الصندوق
إلغاء سندات قبض أو مصاريف
إدارة مبالغ لحساب الموكل
مسح بيانات
استعادة نسخ احتياطية
```

---

## 6. منع تصعيد الصلاحيات

هذه قاعدة تقنية وإدارية إلزامية:

```text
أي مستخدم لا يستطيع منح صلاحية لا يملكها.
أي مستخدم لا يستطيع إنشاء دور أعلى من دوره.
أي مستخدم لا يستطيع تعديل دور أعلى أو مساوٍ له.
أي مستخدم لا يستطيع تعيين مستخدم بدور أعلى أو مساوٍ له.
```

أمثلة:

```text
مدير المكتب لا يستطيع تعديل مالك المكتب.
مدير المكتب لا يستطيع إنشاء مدير مكتب جديد.
مدير المكتب لا يستطيع منح صلاحيات مالية حساسة إذا لم تكن لديه.
محامي أستاذ لا يستطيع إدارة الأدوار إلا إذا منحه المالك صلاحية خاصة، وهذا غير مستحسن افتراضياً.
```

---

## 7. الصلاحيات المالية

المالية هي أكثر محاور التطبيق حساسية، لذلك تقسم إلى مستويات.

### 7.1 عرض مالي

```text
عرض ملخص مالي
عرض مالية ملف
عرض ذمم الموكلين
عرض التقارير المالية
```

يمكن للمالك منح هذه الصلاحيات لمدير المكتب أو محامي أستاذ حسب سياسة المكتب.

### 7.2 إدخال مالي عادي

```text
إضافة اتفاق أتعاب
إضافة سند قبض
إضافة مصروف
إضافة مبلغ لحساب الموكل
طباعة سند قبض
```

يمكن منحها لمدير المكتب أو محامي أستاذ عند الحاجة، مع تسجيل كامل في سجل الأحداث.

### 7.3 إدارة مالية حساسة

تبقى افتراضياً للمالك فقط:

```text
إلغاء سند قبض
إلغاء مصروف
تصحيح قيد مالي
عكس قيد
إدارة الصندوق
نقل بين الصناديق
تعديل إعدادات مالية
إدارة مبالغ لحساب الموكل
تسليم مبلغ لحساب الموكل
اقتطاع أتعاب من مبلغ لحساب الموكل
تصدير تقارير مالية شاملة
```

يمكن للمالك منح بعضها لمدير المكتب إذا أراد، لكن لا تكون ممنوحة افتراضياً.

---

## 8. مبالغ لحساب الموكل

مبالغ لحساب الموكل أخطر من الأتعاب لأنها ليست أموال المكتب.

صلاحياتها مستقلة:

```text
عرض مبالغ لحساب الموكل
إضافة مبلغ لحساب الموكل
تسليم مبلغ للموكل
اقتطاع أتعاب من مبلغ لحساب الموكل
إلغاء / تصحيح حركة مبلغ لحساب الموكل
```

الافتراضي:

```text
المالك فقط.
```

يمكن للمالك منح مدير المكتب عرضاً فقط، أو إضافة فقط، دون تسليم أو اقتطاع، حسب سياسة المكتب.

---

## 9. صندوق المكتب

صلاحيات الصندوق:

```text
عرض صندوق المكتب
إضافة حركة صندوق
تحويل بين الصناديق
تعديل / إلغاء حركة صندوق
إدارة الصناديق والحسابات
```

الافتراضي:

```text
المالك يدير الصندوق.
مدير المكتب يمكنه العرض أو إضافة مصروف عادي إذا سمح المالك.
```

---

## 10. الصلاحيات الإدارية الحساسة

تبقى للمالك افتراضياً:

```text
إدارة الأدوار
تعديل صلاحيات الأدوار
تعيين مالك مكتب
تعيين مدير مكتب
تعطيل مستخدم
مسح بيانات التشغيل
استعادة نسخة احتياطية
تصدير كامل البيانات
تعديل بيانات المكتب الأساسية الحساسة
إدارة إعدادات الترخيص أو التفعيل عند إضافتها لاحقاً
```

مدير المكتب يمكنه، إذا سمح المالك، فقط:

```text
إضافة مستخدم محامي أستاذ
تعطيل مستخدم أدنى منه
تعديل بيانات مستخدم أدنى منه
```

---

## 11. المكاتب الصغيرة والشركات الكبيرة

في المكتب الصغير غالباً:

```text
مالك المكتب = المحامي نفسه
```

وقد لا يحتاج مدير مكتب.

في الشركات الكبيرة يمكن للمالك إنشاء:

```text
مدير مكتب
عدة محامين أساتذة
```

لكن تبقى صلاحيات السيادة عند مالك المكتب.

يمكن مستقبلاً إضافة أدوار مثل:

```text
شريك / مدير شريك
مجلس إدارة
```

لكنها ليست ضمن المرحلة الحالية.

---

## 12. الخلاصة التنفيذية

القواعد المعتمدة:

```text
مالك المكتب = كل الصلاحيات.
مدير المكتب = إدارة تشغيلية، ولا يستطيع تعيين مالك أو مدير.
مدير المكتب يستطيع تعيين وإدارة ما دون المدير فقط.
الصلاحيات المالية الحساسة تبقى للمالك افتراضياً.
مبالغ لحساب الموكل والصندوق والإلغاء المالي تبقى للمالك افتراضياً.
لا يسمح لأي مستخدم بمنح صلاحيات لا يملكها.
لا يسمح لأي مستخدم بتعديل دور أعلى أو مساوٍ له.
لا يجوز حذف أو تعطيل آخر مالك مكتب.
```
