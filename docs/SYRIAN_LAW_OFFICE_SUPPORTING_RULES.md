# قواعد داعمة لهيكل مكتب المحامي السوري

تاريخ الاعتماد: 2026-07-18

هذه الوثيقة تثبت القواعد الداعمة التي ظهرت بعد مطابقة الهيكل المتفق عليه مع واقع عمل المحامي السوري وأفضل ممارسات إدارة الملفات القانونية.

هذه القواعد لا تلغي ما تم اعتماده سابقاً، بل تضبطه وتمنع أن يتحول التطبيق إلى CRUD عام أو إلى أرشيف منفصل عن العمل اليومي.

---

## 1. القاعدة العامة

الهيكل المعتمد يبقى:

```text
ملف مكتب موحد
جاري / منتهٍ
إدخال أرشيف قديم منفصل
Workspace متخصص حسب نوع الملف
مستندات ومالية وأوامر عمل وسجل أحداث مرتبطة بالملف
```

والإضافات المعتمدة هي:

```text
فحص تعارض مصالح / وجود طرف سابق
الوكالة ككيان مستقل متعدد الارتباطات
محضر تسليم مستندات
Checklist إغلاق إداري
توسيع القضاء الإداري والقوائم المرجعية دون تعقيد الواجهة
المستندات كطبقة داخل الملفات لا كقسم جانبي مستقل
```

---

## 2. فحص تعارض مصالح / وجود طرف سابق

### 2.1 الهدف

قبل فتح ملف جديد، يجب أن يساعد التطبيق المكتب على معرفة:

```text
هل الموكل موجود سابقاً؟
هل الخصم كان موكلاً سابقاً؟
هل توجد دعوى أو عقد أو إجراء أو وكالة أو شركة مرتبطة بنفس الطرف؟
هل يوجد ملف قديم بين نفس الأطراف؟
هل يوجد تعارض محتمل يحتاج قرار المحامي؟
```

التطبيق لا يعطي فتوى قانونية ولا يمنع المحامي تلقائياً، بل ينبه ويطلب قراراً موثقاً عند وجود تعارض قوي.

### 2.2 أين يظهر؟

يظهر في:

```text
مكتب العمل > إنشاء جديد
```

خصوصاً بعد إدخال الأطراف وقبل الحفظ النهائي.

في الأرشيف القديم، يظهر الفحص كأداة ربط واكتشاف ملفات مشابهة، لا كمانع إدخال.

### 2.3 بيانات البحث

يعتمد الفحص على:

```text
اسم الموكل
اسم الخصم
رقم الهاتف
الرقم الوطني
السجل التجاري أو رقم الجهة
اسم الجهة
اسم المحامي السابق إن وجد
```

ويبحث في:

```text
الأشخاص والجهات
الدعاوى
الإجراءات / المعاملات
العقود
الشركات
الوكالات
الأرشيف القديم
```

### 2.4 نتائج الفحص

النتائج تكون بدرجات:

```text
لا يوجد تطابق
تطابق ضعيف
تطابق محتمل
تطابق قوي
تعارض محتمل
```

أمثلة:

```text
لا يوجد تطابق: لا توجد ملفات سابقة لهذا الطرف.
تطابق محتمل: يوجد شخص باسم مشابه في ملف سابق.
تطابق قوي: نفس الرقم الوطني أو الهاتف أو السجل.
تعارض محتمل: الخصم الحالي كان موكلاً سابقاً في ملف آخر.
علاقة سابقة لا تمنع: يوجد عقد سابق لنفس الموكل.
```

### 2.5 قرار المستخدم

عند ظهور نتيجة مهمة، تظهر خيارات:

```text
متابعة
مراجعة الملفات المرتبطة
تسجيل ملاحظة تعارض
إيقاف فتح الملف
```

إذا كان التعارض قوياً، يطلب سبب متابعة إلزامي، مثل:

```text
تمت مراجعة الملف السابق ولا يوجد تعارض.
الموضوع مختلف ولا توجد معلومات سرية مؤثرة.
قرار المالك / المدير بالمتابعة.
سبب آخر.
```

### 2.6 الصلاحية

في المرحلة الأولى:

```text
الفحص تحذيري لا يمنع الحفظ تلقائياً.
التعارض القوي يتطلب قراءة التحذير وكتابة سبب متابعة.
يمكن لاحقاً إضافة إعداد يجعل المتابعة بعد تعارض قوي محصورة بالمالك أو المدير.
```

### 2.7 سجل الأحداث

إذا وجد تعارض محتمل واستمر المستخدم، يسجل في سجل الأحداث:

```text
تم إجراء فحص وجود طرف سابق / تعارض محتمل.
تمت المتابعة رغم وجود تطابق / تعارض محتمل مع ذكر السبب.
```

## 3. الوكالة ككيان مستقل متعدد الارتباطات

### 3.1 الهدف

الوكالة القضائية في واقع المكتب السوري ليست مجرد صورة داخل دعوى، بل كيان قانوني وإداري مستقل يمكن أن يخدم أكثر من ملف.

قد ترتبط الوكالة بـ:

```text
دعوى
إجراء / معاملة
عقد
شركة
تنفيذ
أكثر من ملف
```

وقد يحصل عليها لاحقاً:

```text
عزل
اعتزال
انتهاء استعمال
استبدال بصورة مصدقة
تصحيح بيانات
```

### 3.2 بيانات الوكالة

كل وكالة يجب أن تحتوي مبدئياً:

```text
رقم داخلي للوكالة داخل المكتب
نوع سند التوكيل
رقم سجل الوكالة
الرقم الأبيض
تاريخ ووقت التنظيم
فرع النقابة
مندوب الوكالات
هاتف المندوب
الموكلون المشمولون بالوكالة
المحامون الوكلاء
نطاق الوكالة / صلاحياتها المهمة
صورة سند الوكالة
حالة الوكالة
ملاحظات
```

أنواع سند التوكيل الأساسية:

```text
سند توكيل عام
سند توكيل خاص
سند توكيل خاص شرعي
```

### 3.3 حالة الوكالة

الحالات المعتمدة:

```text
فعالة
منتهية
ملغاة
معزول عنها
اعتزال من الوكالة
غير مكتملة البيانات
```

### 3.4 ارتباطات الوكالة

الوكالة يمكن أن ترتبط بـ:

```text
دعوى واحدة أو أكثر
إجراء واحد أو أكثر
عقد واحد أو أكثر
شركة واحدة أو أكثر
ملف وكالة مستقل
```

ويجب أن يظهر في ملف الوكالة قسم:

```text
الملفات المرتبطة
```

يعرض كل ملف يستخدم هذه الوكالة.

### 3.5 إضافة وكالة داخل ملف

عند إضافة وكالة داخل دعوى أو إجراء أو عقد أو شركة، تكون الخيارات:

```text
اختيار وكالة موجودة
إضافة وكالة جديدة
إضافة صورة وكالة دون بيانات كاملة
سأضيفها لاحقاً
```

الأثر:

```text
اختيار وكالة موجودة: يربط الملف بالوكالة دون تكرار بياناتها.
إضافة وكالة جديدة: ينشئ كيان وكالة ويربطه بالملف.
إضافة صورة دون بيانات كاملة: يحفظ المستند ويضع حالة الوكالة غير مكتملة البيانات.
سأضيفها لاحقاً: يضيف نقصاً تحذيرياً في الملف الجاري.
```

### 3.6 العزل والاعتزال

يجب أن يحتوي ملف الوكالة زر:

```text
تسجيل عزل / اعتزال
```

الحقول:

```text
نوع الحدث: عزل / اعتزال
تاريخ الحدث
سبب الحدث
مستند العزل / الاعتزال
هل يؤثر على الملفات المرتبطة؟
ملاحظات
```

إذا كانت الوكالة مرتبطة بملفات جارية، يظهر تحذير:

```text
هذه الوكالة مرتبطة بملفات جارية. يجب مراجعة أهلية المتابعة أو إضافة وكالة بديلة.
```

### 3.7 الأثر على الملفات المرتبطة

عند تسجيل عزل أو اعتزال، يجب أن يظهر داخل الملفات الجارية المرتبطة نقص أو تنبيه:

```text
الوكالة المرتبطة بالملف لم تعد فعالة.
```

ولا يغلق الملف تلقائياً، بل يطلب من المستخدم اتخاذ إجراء.

## 4. المستندات كطبقة داخل الملفات

### 4.1 القرار

لا يعاد إنشاء قسم جانبي مستقل باسم:

```text
المستندات الرقمية
الأرشيف الفيزيائي
```

لأن ذلك يشتت المستخدم ويفصل المستند عن الملف القانوني.

المستندات تظهر في:

```text
تبويب المستندات داخل كل ملف
إدخال الأرشيف القديم
التقارير والكشوف
البحث العام
```

### 4.2 بيانات المستند

كل مستند يجب أن يدعم:

```text
نوع المستند
عنوان المستند
الملف المرتبط
المرحلة المرتبطة إن وجدت
الجلسة المرتبطة إن وجدت
تاريخ المستند
تاريخ الإضافة
من أضافه
هل له أصل ورقي؟
مكان الأصل الورقي
حالة الأصل الورقي
ملاحظات
```

### 4.3 حالة الأصل الورقي

الحالات:

```text
موجود في المكتب
مسلم للموكل
مسلم لمحامٍ آخر
مفقود
غير معروف
لا يوجد أصل ورقي
```

### 4.4 مكان الأصل الورقي

الحقول:

```text
مكان الحفظ
الخزانة
الرف
الصندوق
رقم الغلاف / المصنف
ملاحظات مكان الحفظ
```

### 4.5 البحث والتقارير

رغم عدم وجود قسم جانبي مستقل، يجب أن يستطيع المستخدم لاحقاً البحث مركزياً في المستندات من:

```text
البحث العام
التقارير والكشوف
داخل ملف محدد
```

## 5. محضر تسليم مستندات

### 5.1 الهدف

عند انتهاء علاقة المكتب بالملف أو تسليم الأصول، يجب توثيق ما تم تسليمه ولمن ومتى.

### 5.2 متى يظهر؟

يظهر في حالات:

```text
التخلي عن الملف
تسليم الملف لمحامٍ آخر
إغلاق نهائي مع إعادة أصول
طلب الموكل استلام أوراقه
إصدار لا مانع
إغلاق إداري يتطلب تسليم مستندات
```

### 5.3 الحقول

```text
تاريخ التسليم
المستلم
صفة المستلم
رقم هاتف المستلم
طريقة التحقق من الهوية
المستندات المسلمة
هل بقيت نسخ في المكتب؟
ملاحظات
توقيع المستلم إن أمكن
مرفق صورة المحضر
```

### 5.4 صفة المستلم

```text
الموكل
وكيل الموكل
محامٍ جديد
شخص مفوض
جهة رسمية
أخرى
```

### 5.5 أثر الحفظ

بعد حفظ محضر التسليم:

```text
تحدث حالة المستندات المسلمة إلى مسلم للموكل أو مسلم لمحامٍ آخر أو الحالة المناسبة.
يضاف المحضر إلى تبويب المستندات.
يسجل الحدث في سجل الأحداث.
```

نص سجل الأحداث المقترح:

```text
تم تسليم مستندات الملف إلى ... بتاريخ ...
```

## 6. Checklist الإغلاق الإداري

### 6.1 الهدف

الإغلاق القضائي أو العملي لا يكفي وحده. يجب التأكد من أن الملف أغلق إدارياً بشكل منظم.

الغرض منع نسيان:

```text
الأتعاب
المصاريف
الأصول الورقية
لا مانع
تسليم الأوراق
أوامر العمل المفتوحة
الأرشفة
```

### 6.2 أين يظهر؟

يظهر في:

```text
ويزارد إغلاق / إنهاء مرحلة
```

عند اختيار نتائج مثل:

```text
إغلاق نهائي
التخلي عن الملف / تسليمه
انتهاء التنفيذ
صلح / تسوية
تنازل / إسقاط
```

كما يمكن أن يظهر في الدعوى أو الملف المنتهي كتقسيم:

```text
مراجعة الإغلاق الإداري
```

### 6.3 عناصر Checklist

```text
تم تحديد سبب الإغلاق
تم إرفاق القرار / المستند الأساسي
تم تحديث الحالة المالية
تم تسجيل المصاريف المتبقية
تم إصدار سند قبض إن وجد
تم تسليم الأصول للموكل أو تحديد مكانها
تم إنشاء لا مانع عند التخلي إذا اختير ذلك
تم إنشاء محضر تسليم مستندات عند الحاجة
تم تحديد مكان الأصل الورقي
تم إغلاق أوامر العمل المفتوحة أو تحويلها لأوامر لاحقة
تم تسجيل ملاحظة الإغلاق
```

### 6.4 هل تمنع الإغلاق؟

القاعدة:

```text
لا تمنع الإغلاق دائماً.
تعرض مراجعة واضحة.
بعض العناصر تصبح إلزامية حسب سبب الإغلاق.
```

أمثلة إلزامية:

```text
في التخلي: سبب التخلي إلزامي، وحالة تسليم المستندات إلزامية.
في التنازل عن الحق: تحذير إلزامي، ومستند أو سبب واضح، وصلاحية أعلى.
في انتهاء التنفيذ: نتيجة التنفيذ أو سبب تعذره إلزامي.
```

### 6.5 أثر الحفظ

عند حفظ Checklist:

```text
تسجل في سجل الأحداث.
تظهر في تبويب إجراءات لاحقة / طعون أو الملخص حسب حالة الملف.
تساعد في التقارير على تمييز ملفات منتهية إدارياً بالكامل عن ملفات منتهية قضائياً فقط.
```

## 7. القضاء الإداري والقوائم المرجعية

### 7.1 القاعدة

القضاء الإداري يبقى تصنيفاً مستقلاً، ولا يدمج داخل المدني.

القائمة الأساسية في الواجهة تبقى بسيطة:

```text
قضاء إداري
```

وتحتها:

```text
محكمة عمالية
محكمة قضاء إداري
المحكمة الإدارية العليا
محكمة مسلكية
جهة إدارية أخرى
```

### 7.2 قوائم مرجعية قابلة للتوسع

في القوائم المرجعية يمكن دعم خيارات إضافية مثل:

```text
المحاكم الإدارية
هيئة مفوضي الدولة
دائرة توحيد المبادئ
إدارة التفتيش
جهات إدارية أخرى
```

لا تظهر كل هذه الخيارات بالضرورة في الواجهة الأولى حتى لا تتعقد على المستخدم، لكنها تكون قابلة للإضافة أو التفعيل.

### 7.3 القاعدة العامة للقوائم

```text
قائمة افتراضية سورية
إضافة خيار مخصص
تعطيل خيار دون حذفه عند وجود بيانات مرتبطة
تعديل آمن للأسماء عند الحاجة
تمييز الخيارات النظامية عن الخيارات المخصصة
```

## 8. أثر هذه القواعد على الشاشات القادمة

### 8.1 شاشة الملفات الجارية

يجب أن تعرض مؤشرات مثل:

```text
تعارض محتمل غير محسوم
وكالة غير مكتملة
وكالة معزول عنها / اعتزال
مستندات أصلية غير محددة المكان
أوامر عمل مفتوحة
نواقص تشغيلية
```

### 8.2 شاشة الملفات المنتهية

يجب أن تعرض:

```text
هل الإغلاق الإداري مكتمل؟
هل يوجد محضر تسليم مستندات؟
هل توجد أصول ورقية لم تسلم أو لم يحدد مكانها؟
هل توجد مالية متبقية؟
هل توجد إجراءات لاحقة متاحة؟
```

### 8.3 ويزارد إنشاء دعوى / ملف جديد

يجب أن يحتوي أو يدعم:

```text
فحص وجود طرف سابق / تعارض محتمل
اختيار وكالة موجودة أو إضافة وكالة جديدة
عرض نواقص تحذيرية قبل الحفظ
```

### 8.4 ويزارد الأرشيف القديم

يجب أن يدعم:

```text
ربط أطراف بملفات سابقة عند وجود تشابه
تسجيل الأصل الورقي
ربط أو إنشاء وكالة قديمة
إدخال مستندات وتصنيفها لاحقاً
```

### 8.5 ويزارد الإغلاق

يجب أن يدعم:

```text
Checklist إغلاق إداري
محضر تسليم مستندات
لا مانع
تحديث حالة المستندات والأصل الورقي
```

## 9. خلاصة تنفيذية

القواعد المعتمدة:

```text
لا يفتح ملف جديد دون فحص تحذيري لوجود أطراف سابقة أو تعارض محتمل.
الوكالة كيان مستقل قابل للربط بعدة ملفات.
العزل والاعتزال أحداث مؤثرة على الوكالة والملفات الجارية المرتبطة بها.
المستندات تبقى داخل الملفات، مع بحث وتقارير مركزية لاحقاً.
محضر تسليم المستندات جزء من التخلي والإغلاق الإداري.
Checklist الإغلاق الإداري تميز بين إغلاق قضائي وإغلاق مكتبي كامل.
القضاء الإداري يبقى مستقلاً، والقوائم المرجعية قابلة للتوسع دون تعقيد الواجهة.
```

هذه القواعد يجب أخذها بعين الاعتبار قبل تنفيذ شاشات الملفات الجارية والمنتهية وقبل بناء ويزاردات الملفات الأخرى.
