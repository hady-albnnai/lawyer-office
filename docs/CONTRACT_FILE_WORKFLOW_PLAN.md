# خطة ملف العقد وويزارد العقود

تاريخ الاعتماد: 2026-07-20

هذه الخطة تضبط فلسفة ملف العقد داخل التطبيق، وتركز على أن التطبيق لا يعيد كتابة تفاصيل العقد القانونية داخل شاشات إدخال كثيرة، بل يدير ملف العقد وفهرسته ومتابعته، بينما تبقى تفاصيل العقد داخل نموذج العقد أو المستند المرفق.

---

## 1. القاعدة الأساسية

ملف العقد في التطبيق هو:

```text
غلاف إداري / قانوني للعقد
فهرسة وبحث
متابعة ومواعيد
مستندات ونسخ
أطراف
ملفات مرتبطة
مالية
سجل أحداث
```

وليس بديلاً عن نص العقد نفسه.

لذلك لا يطلب التطبيق من المستخدم إعادة إدخال كامل مضمون العقد داخل الواجهة.

---

## 2. ما الذي يحفظه التطبيق عن العقد؟

يحفظ التطبيق بيانات الفهرسة والمتابعة اللازمة فقط:

```text
رقم ملف المكتب
نوع ملف العقد
التصنيف القانوني
التصنيف الفرعي
أطراف العقد
تاريخ العقد
تاريخ فتح ملف العقد
حالة العقد
نسخة العقد / النموذج / الملف
ملاحظات مختصرة للبحث عند الحاجة
الملفات المرتبطة
المستندات
أوامر العمل
المالية
سجل الأحداث
```

أما تفاصيل العقد مثل:

```text
الثمن
الشروط التفصيلية
أوصاف محل العقد
بنود الضمان
بنود الفسخ
بنود التسليم
الغرامات والجزاءات
الالتزامات الفنية أو المالية الدقيقة
```

فتبقى داخل:

```text
نموذج Word
ملف PDF
المستند المرفق
النص الذي يكتبه المحامي خارج التطبيق
```

ولا يطلبها التطبيق كحقول مستقلة مكررة.

---

## 3. رقم ملف العقد

رقم ملف المكتب للعقد يولد تلقائياً حسب قاعدة ترقيم ملفات المكتب:

```text
عقد/2026/0001
```

ولا يكون قابلاً للتعديل بعد الحفظ.

---

## 4. بداية ويزارد العقد

يبدأ ويزارد العقد من:

```text
مكتب العمل > إنشاء جديد > عقد
```

ولا يبدأ من شاشة الملفات الجارية أو المنتهية.

الأرشيف القديم للعقود يبدأ من:

```text
ملفات المكتب > إدخال الأرشيف القديم
```

---

## 5. الخطوة الأولى — نوع ملف العقد

العنوان:

```text
ماذا تريد أن تفعل؟
```

الخيارات:

```text
إنشاء عقد جديد
مراجعة / تعديل عقد جاهز
متابعة عقد موقع
```

### 5.1 إنشاء عقد جديد

الإيضاح:

```text
استخدم هذا الخيار إذا كان المكتب سيبدأ بصياغة عقد جديد من الصفر أو من نموذج جاهز.
```

### 5.2 مراجعة / تعديل عقد جاهز

الإيضاح:

```text
استخدم هذا الخيار إذا كان لديك عقد أو مسودة موجودة وتريد مراجعتها أو تعديلها أو إعداد نسخة معدلة عنها.
```

### 5.3 متابعة عقد موقع

الإيضاح:

```text
استخدم هذا الخيار إذا كان العقد موقعاً بالفعل، والمكتب سيتابع التزاماته أو إجراءاته أو نزاعاً ناشئاً عنه.
```

ملاحظة:

```text
لا تستخدم عبارة تعديل عقد قديم داخل إنشاء جديد، حتى لا تختلط مع إدخال الأرشيف القديم.
```

---

## 6. الخطوة الثانية — التصنيف القانوني

بعد اختيار نوع ملف العقد، يختار المستخدم التصنيف القانوني.

القائمة الأساسية:

```text
عقود واردة على الملكية
عقود واردة على الانتفاع
عقود واردة على العمل / الخدمة
عقود صلح / تسوية
عقود شركات / شراكات
عقود ضمان / كفالة / رهن
وكالات / تفويضات
عقود تجارية
عقود أخرى
```

الغرض من التصنيف القانوني:

```text
تنظيم النماذج
تسهيل البحث والفلترة
إظهار التصنيفات الفرعية المناسبة
ربط العقد بالنماذج السورية المعتمدة
```

ولا يعني أن التطبيق سيطلب تفاصيل قانونية كاملة لكل تصنيف داخل الواجهة.

---

## 7. الخطوة الثالثة — التصنيف الفرعي

بعد اختيار التصنيف القانوني، تظهر الأنواع الفرعية المناسبة.

### 7.1 عقود واردة على الملكية

```text
بيع عقار
بيع منقول
بيع مركبة
بيع حصة / أسهم
مقايضة
هبة
قرض / دين
```

### 7.2 عقود واردة على الانتفاع

```text
إيجار سكني
إيجار تجاري
إيجار مكتب / مهنة
إيجار أرض
عارية استعمال
```

### 7.3 عقود واردة على العمل / الخدمة

```text
مقاولة
عقد عمل
عقد خدمات
عقد استشارة
توريد مع تنفيذ
```

### 7.4 عقود صلح / تسوية

```text
صلح قضائي
صلح غير قضائي
تسوية نزاع
مخالصة / إبراء
إنهاء علاقة تعاقدية
```

### 7.5 عقود شركات / شراكات

```text
اتفاق شركاء
عقد محاصة
عقد تأسيس شركة
ملحق عقد تأسيس
تنازل عن حصص
قرار شركاء / هيئة عامة
```

### 7.6 وكالات / تفويضات

```text
تفويض خاص
عقد وكالة مدنية
وكالة بيع
وكالة إدارة
وكالة غير قابلة للعزل مرتبطة بحق
```

### 7.7 عقود تجارية

```text
توريد
توزيع
وكالة تجارية
استثمار
تعاون تجاري
استعمال علامة تجارية
سمسرة / وساطة
```

### 7.8 عقود أخرى

```text
عقد آخر
تصنيف مخصص
```

القوائم قابلة للتوسع من القوائم المرجعية، لكن لا يجوز حذف خيار مستخدم سابقاً، بل يعطل عند الحاجة.

---

## 8. الخطوة الرابعة — طريقة إنشاء العقد

بعد اختيار التصنيف القانوني والفرعي، يختار المستخدم طريقة إنشاء العقد:

```text
اختيار نموذج جاهز
إدخال عقد / نموذج للتعديل
كتابة عقد يدوياً
```

### 8.1 اختيار نموذج جاهز

الإيضاح:

```text
اختر من نماذج العقود السورية الجاهزة داخل التطبيق، ثم عدّل النموذج كمستند عقد حسب الحالة.
```

هذا الخيار يعتمد على مكتبة نماذج مصنفة حسب:

```text
التصنيف القانوني
التصنيف الفرعي
نوع النموذج
```

مثال:

```text
عقود واردة على الملكية > بيع عقار > نموذج عقد بيع عقار
```

### 8.2 إدخال عقد / نموذج للتعديل

الإيضاح:

```text
ارفع عقداً أو مسودة Word/PDF موجودة، ثم احفظها ضمن ملف العقد مع إمكانية إعداد نسخة معدلة عنها.
```

يحفظ التطبيق:

```text
النسخة الأصلية
النسخة المعدلة عند وجودها
ملاحظات المراجعة إن وجدت
```

### 8.3 كتابة عقد يدوياً

الإيضاح:

```text
ابدأ من ملف Word فارغ أو قالب بسيط، واكتب العقد بنفسك خارج التطبيق، ثم اربطه بملف العقد.
```

لا يبنى محرر عقود داخلي معقد في المرحلة الأولى.

---

## 9. أطراف العقد

الأطراف هي أهم بيانات العقد التي يحتاجها التطبيق للبحث والربط وفحص التعارض.

القاعدة:

```text
العقد يحتوي عدداً غير محدود من الأطراف.
كل طرف يحتوي عدداً غير محدود من الأشخاص أو الجهات.
كل شخص / جهة داخل الطرف يمكن أن يكون له صفة.
```

الحد الأدنى المطلوب:

```text
اسم الطرف داخل العقد
الأشخاص / الجهات ضمن الطرف
صفة كل شخص / جهة إن وجدت
```

أمثلة:

```text
الطرف الأول: البائعون
- أحمد
- خالد

الطرف الثاني: المشترون
- محمد
```

أمثلة أسماء الأطراف:

```text
البائع
المشتري
المؤجر
المستأجر
الفريق الأول
الفريق الثاني
الشركاء
المقاول
صاحب العمل
العامل
المتنازل
المتنازل له
الضامن
الكفيل
الممول
طرف آخر
```

لا يطلب التطبيق تفاصيل بنود العلاقة بين الأطراف، لأنها تبقى داخل النموذج.

## 11. نسخة العقد والمستندات

ملف العقد يجب أن يدعم:

```text
نموذج جاهز مختار
ملف Word
ملف PDF
صورة عقد
نسخة أصلية
نسخة معدلة
نسخة موقعة
مستندات مرفقة
```

في حال عدم إرفاق نسخة عقد، يظهر نقص تحذيري:

```text
لا توجد نسخة عقد مرفقة.
```

## 12. النماذج الجاهزة المعتمدة

### 12.1 مصدر النماذج

تعتمد نماذج الأستاذ المحامي حازم الجزار كمرجع أساسي للنماذج الجاهزة في التطبيق.

توجد مراجع تشير إلى كتاب:

```text
صديق المحامي
```

وأنه يتضمن نماذج الدعاوى والأعمال الإدارية التي يحتاجها المحامي في عمله، وأنه من إعداد المستشار المحامي حازم الجزار.

كما تشير مراجع أخرى إلى أن برنامج / مرجع `الشامل في القانون` أضيف إليه كتاب `صديق المحامي` ويحتوي على صيغ أصولية من إعداد المحامي الأستاذ حازم الجزار.

### 12.2 قاعدة الاعتماد

نماذج الجزار معتمدة كمصدر أولي للنماذج.

لكن قبل إدخال أي نموذج نهائياً في التطبيق يجب أن يمر بمراجعة قانونية داخلية:

```text
مراجعة نص النموذج.
مطابقة النموذج مع القانون السوري النافذ.
مراجعة المواد القانونية اللازمة.
تحديد ما إذا كان النموذج يحتاج تحديثاً أو تعديلاً.
تعديل النموذج عند الحاجة.
عرض النسخة المعدلة على المحامي للمراجعة والاعتماد.
```

لا يعتمد أي نموذج للاستخدام النهائي قبل:

```text
مراجعة قانونية
تعديل عند الحاجة
اعتماد المحامي
```

### 12.3 عدم النسخ العشوائي

لا تعتمد نماذج عشوائية من الإنترنت كما هي.

القاعدة:

```text
نماذج الجزار هي الأساس.
أي نموذج يراجع ويطابق مع القانون السوري قبل إدخاله.
أي تعديل يتم توثيقه.
المحامي يراجع النسخة النهائية قبل اعتمادها.
```

## 14. الملفات المرتبطة بالعقد

ملف العقد يمكن أن يرتبط بـ:

```text
دعوى
إجراء / معاملة
شركة
وكالة
عقد آخر
```

أمثلة:

```text
عقد بيع عقار ↔ إجراء فراغ عقاري
عقد بيع عقار ↔ دعوى تثبيت بيع
عقد تأسيس / اتفاق شركاء ↔ ملف شركة
عقد أصلي ↔ ملحق عقد
عقد صلح ↔ دعوى منتهية بصلح
```

الربط لا يدمج الملفات، بل يسجل علاقة واضحة بين الملفات.

## 18. إغلاق / إنهاء ملف العقد

أسباب الإغلاق:

```text
تم توقيع العقد وانتهى دور المكتب
تم توقيع العقد مع متابعة لاحقة
تم تنفيذ العقد بالكامل
تم إلغاء العقد
فشل التفاوض
انسحاب أحد الأطراف
انتهت مدة العقد
تحول إلى نزاع / دعوى
سبب آخر
```

إذا تحول إلى نزاع، يظهر زر:

```text
فتح دعوى مرتبطة
```

الإيضاح:

```text
استخدمه إذا نشأ نزاع من العقد وتقرر فتح دعوى.
```

## 19. النواقص التحذيرية في ملف العقد

النواقص تكون مرتبطة بإدارة ملف العقد لا بتفاصيل بنوده.

أمثلة:

```text
لا توجد نسخة عقد مرفقة
لا توجد أطراف كافية
لم يحدد التصنيف القانوني
لم يحدد التصنيف الفرعي
لا يوجد تاريخ عقد
لا توجد نسخة موقعة رغم حالة العقد موقع
أصل ورقي غير محدد
ملف مرتبط مطلوب غير موجود عند الحاجة
```

ولا ينشئ التطبيق نواقص من نوع:

```text
لم تدخل الثمن
لم تدخل شروط التسليم
لم تدخل غرامة التأخير
```

لأن هذه التفاصيل تبقى داخل نموذج العقد.

---

## 20. الخلاصة التنفيذية

القواعد المعتمدة:

```text
ملف العقد في التطبيق غلاف وفهرسة ومتابعة، وليس شاشة لإعادة كتابة مضمون العقد.
تفاصيل العقد تبقى داخل نموذج Word / PDF / المستند المرفق.
التطبيق يحفظ التصنيف القانوني والفرعي، الأطراف، التاريخ، الحالة، النسخة، والملفات المرتبطة.
ويزارد العقد يعتمد: نوع ملف العقد، التصنيف القانوني، التصنيف الفرعي، طريقة إنشاء العقد.
طرق إنشاء العقد: اختيار نموذج جاهز، إدخال عقد / نموذج للتعديل، كتابة عقد يدوياً.
نماذج حازم الجزار معتمدة كمصدر أولي للنماذج الجاهزة.
كل نموذج يراجع ويطابق مع القانون السوري ويعدل عند الحاجة قبل الاعتماد النهائي.
المحامي يراجع النسخة النهائية للنموذج قبل اعتمادها داخل التطبيق.
```
