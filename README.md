# 🏛️ تطبيق إدارة وأرشفة مكتب محاماة سوري (V6.2 Production Blueprint)

> **نظام رقمي متكامل وشامل لإدارة الدعاوى القضائية، الشركات، العقود، والوكالات في مكتب المحاماة السوري.**  
> **التقنية المعتمدة:** `Flutter` + `Drift (SQLite / SQLCipher)` + `Riverpod` + `GoRouter`  
> **البيئة المستهدفة حالياً:** `Windows` (Offline-First أولاً، جاهز للربط الشبكي والتوسع إلى `Android` لاحقاً).  
> **اللغة والواجهة:** عربية بالكامل مع دعم `RTL` الأصلي وخطوط المحاماة الرسمية (`Cairo / Amiri`).

---

## 📚 التوثيق الرسمي ومراجع التطوير

يعتمد هذا المشروع في برمجته وتطويره على القواعد الدستورية والهندسية الموثقة في الملفات التالية:

1. **[🛠️ دستور ودليل التطوير (DEVELOPMENT_GUIDELINES.md)](./DEVELOPMENT_GUIDELINES.md):**  
   الدستور الإلزامي للمطور (الوكيل الذكي) الذي يحكم دورة حياة التعديل البرمجي المكونة من 3 مراحل (البرمجة ← التوثيق ← الرفع)، ويطبق **مبدأ الدقة المتناهية** والمحظورات التقنية المنسوخة والمحفوظة من مستودع العقارات (`sweeda-real-estate-office-elc`).

2. **[📋 الخطة التنفيذية المفصلة (docs/EXECUTION_PLAN.md)](./docs/EXECUTION_PLAN.md):**  
   خارطة الطريق الشاملة ومصفوفة التتبع المفصلة لبناء النظام عبر 9 مراحل تشغيلية وهندسية دقيقة، تغطي 22 جدول قاعدة بيانات في `Drift`، محركات الترقيم السنوي (`2026/001`)، نظام أتمتة المهام اليومية والخط الزمني، نظام النواقص التلقائي، والنسخ الاحتياطي الأسبوعي الذكي المشفر.

---

## ⭐ المبادئ التشغيلية الأساسية للمشروع

- **دورة الحياة الموحدة:** كل كيان تشغيلي يمر عبر الحالات الخمس الموحدة: (`scheduled` ← `in_progress` ← `completed` / `postponed` / `cancelled`).
- **الترحيل التلقائي للمواعيد:** إنجاز الجلسة أو الإجراء يرحّل تلقائياً الموعد القادم إلى الأعمال اليومية للمحامي أو الموظف المسؤول.
- **نظام النواقص التلقائي (`deficiencies`):** إنشاء ملف دون موعد أو دون مستند جوهري يولد إشعار نقص فوري يتم إغلاقه بمجرد استكمال البيانات.
- **تكامل نماذج Word والطباعة القانونية:** دعم فتح نماذج العقود بصيغة `.docx` للتحرير الخارجي والداخلي، وتصدير التقارير الرسمية والمذكرات بتنسيق PDF سوري رسمي.

## 🚀 تعليمات التشغيل والبناء (قيد البدء بالمرحلة 0)

```bash
# جلب الحزم والتبعيات
flutter pub get

# توليد أكواد Drift و Riverpod
dart run build_runner build --delete-conflicting-outputs

# فحص جودة الكود
flutter analyze

# تشغيل التطبيق على بيئة Windows
flutter run -d windows
```
